單一登錄（SSO）是一種會話和用戶身份驗證服務，它允許用戶使用一組登錄憑據（例如，用戶名和密碼）訪問多個應用程序。該服務對已被授予用戶權限的所有應用程序驗證最終用戶，並在用戶在同一會話期間切換應用程序時消除進一步的提示。

Video Cloud Studio提供了基於SAML的單點登錄支持，使發布者可以通過您選擇的身份提供商（IdP）通過SSO訪問Studio。一些功能包括:

  • Video Cloud Studio支持SAML 2.0 SSO協議
  • 身份映射基於用戶名（完整的電子郵件地址），而不只是一個電子郵件域。這為希望某些用戶啟用SSO而其他用戶通過常規Video Cloud Studio登錄的發布者提供了更大的靈活性。
  • 為SSO配置帳戶及其現有用戶後，通過Video Cloud Studio UI添加到該帳戶的新用戶將繼承正在添加帳戶的用戶的SSO設置
  • Video Cloud Studio支持服務提供商發起的SSO登錄。我們目前不支持直接啟動IdP，但是可以提供一個URL，以繞過可以從您的身份提供商門戶鏈接到的Video Cloud Studio登錄屏幕。

有關為您的帳戶啟用SSO的更多信息，請與您的客戶經理聯繫。

好處

為Video Cloud Studio登錄添加SSO的一些好處包括:

  • 一次登錄所有應用程序 -記住多個用戶名和密碼變得難以管理，並且使IT團隊（或工具的帳戶管理員）難以應付。通過啟用SSO並向其添加Video Cloud Studio登錄名，Brightcove帳戶管理員不再需要處理這些問題，該工具的用戶也不必記住Video Cloud Studio的單獨用戶名/密碼。
  • 安全和密碼要求 -企業具有不同的用戶密碼要求，即長度，持續時間和復雜性。使用SSO，Brightcove可以支持這些要求，而不必更改Video Cloud Studio的內置安全性管理來適應每組單獨的要求。
  • 快速輕鬆地禁用用戶訪問 -單點登錄使IT人員可以在員工離開公司後立即集中管理用戶並刪除用戶訪問權限。SSO防止任何心懷不滿的用戶修改或刪除Brightcove媒體，播放器，令牌等。

局限性

當前不支持針對SAML的即時用戶配置。仍然需要通過Studio將用戶添加到帳戶，並且將通過Studio用戶管理UI配置權限/模塊訪問控制。將新用戶添加到帳戶後，他們將繼承要添加他們的用戶的SSO設置。

常問問題

我知道支持基於SAML的SSO，但是兩因素（2FA）或多因素身份驗證又如何呢？支持嗎？

不，我們目前沒有任何計劃在常規Studio登錄中支持2FA。您可以為基於SAML的SSO啟用登錄並將2FA添加到您的IdP設置中，但這將由您的IT團隊在您自己的IdP上進行配置。

是否有計劃支持其他SSO協議（如CAS或Kerberos）？

目前不行。

我們帳戶中有一些用戶不應通過SSO，而其他用戶則需要通過SSO。支持這個嗎？

是的。SSO是按用戶啟用的，因此可以為某些用戶啟用SSO登錄，而為其他用戶啟用常規Studio登錄。需要注意的一件事是，在將新用戶添加到帳戶時，Studio用戶管理UI中無法選擇用戶應通過的身份驗證路徑。當前，用戶將繼承為其添加管理員的配置的任何身份驗證路徑。我們確實打算在以後提供一個在Studio中選擇此選項的選項。現在，針對混合身份驗證流程的解決方法是為SSO配置一名管理員，為Studio常規登錄配置一名管理員。添加新用戶時，管理員應根據應為其設置的身份驗證方法登錄適當的用戶帳戶。

儘管我們支持為不同的身份提供者配置不同的用戶的靈活性，但我們也有一些客戶想要一個單一的IdP，而每個用戶始終為該單一的IdP配置。該配置也是可用的。

是否支持在單個帳戶上為用戶設置多個身份提供者的功能？

是的。我們可以為給定的組織（客戶）設置多個身份提供者，並且可以將用戶分配給任一IdP。將新用戶添加到帳戶時，請參閱上面有關身份驗證路徑繼承的問題。

為用戶打開SAML SSO時，是否允許直接登錄Brightcove的本地登錄名（Studio身份驗證用戶名/密碼）？

否。為用戶啟用SSO後，這是他們可以向其帳戶進行身份驗證的唯一方法。

Brightcove是否支持帶有SP發起的SSO的SAML v2.0？

是的

  • 通過特殊的直接域登錄 URL 支持 SP 啟動的登錄，該 URL 如下所示:
    https://signin.brightcove.com/login/ext/saml?behavior=xxxxxxxxx
  • 用戶也可以轉到signin.brightcove.com頁。當他們在Brightcove登錄頁面上輸入用戶名/密碼時，它們將被重定向到您的IdP進行登錄（如果您尚未登錄）。請注意，在Brightcove登錄頁面上，密碼字段將被忽略-我們僅檢查電子郵件地址以查看是否為用戶啟用了SSO。
Brightcove是否支持在SAMLRequest / SAMLResponse中使用RelayState參數來直接訪問SSO之後的視頻？

否。通過SSO登錄後，用戶將始終被帶到Studio Home儀表板。

SSO集成支持單點註銷嗎？

不。

支持哪些身份提供者？

儘管Brightcove尚未對所有SSO身份提供程序進行測試，但我們相信，只要您的IdP支持SAML 2.0，就不會有任何問題。我們與客戶討論過的一些常見問題包括: Okta，Ping聯合，Ping身份，Microsoft Active Directory，OneLogin和Auth0。
