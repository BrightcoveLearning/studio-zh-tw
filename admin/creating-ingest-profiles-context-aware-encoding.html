創建一個新的上下文感知配置文件

攝取配置文件管理頁面可用於創建新的攝取配置文件。接收配置文件是使用JSON定義的。

要創建新的上下文感知提取配置文件，請執行以下操作:

  1. 點擊行政 Studio標頭中的鏈接。
  2. 點擊攝取個人資料關聯。
  3. 點擊新配置文件>上下文感知。配置文件編輯器將打開。
  4. 進入一個名稱，顯示名稱和描述為個人資料。
  5. 決定是否存儲主文件。存儲母版可以保留母版文件的副本，並且可以在以後無需重新上傳視頻的情況下對視頻進行重新轉碼。有關歸檔主機的其他信息，請參見歸檔大師文檔。
  6. 選擇圖像的設置:
    • 上傳時自動捕獲圖像 -視頻雲將在上傳視頻時捕獲海報和縮略圖圖像。在視頻的50％點處捕獲圖像。
    • 不要捕獲圖像；我將上傳或捕獲自己的圖像 -海報和縮略圖圖像將是被俘或者已上傳使用媒體模塊。
  7. 輸入寬度和高度為了海報和縮圖圖片。
  8. 使用“編解碼器"下拉列表選擇要使用的視頻編解碼器。您可以在H264和HEVC之間進行選擇，也可以使用混合編解碼器配置文件，該配置文件將使用兩個編解碼器（H264，HEVC）創建音色。
  9. 選擇一個最低分辨率和最高解析度通過拖動滑塊。創建演繹時將使用此範圍。
  10. 選擇一個最大比特率。無論內容的複雜程度如何，都不會創建高於此比特率的渲染。較低的比特率可以幫助控制存儲和帶寬方面的成本。
  11. 選擇所需的輸出分辨率。如果所需的分辨率高於源文件的分辨率，則不會創建。
  12. 調整編碼質量控制調整視頻質量和帶寬使用。如果您更關注帶寬使用情況並希望優化以降低帶寬消耗，請向左移動滑塊。如果您更關心質量，並且可以使用更多帶寬來提供更好的圖像質量，可以將滑塊向右移動。
  13. 選擇一個漸進式MP4渲染去創造。要使用“社交"模塊將視頻發佈到社交媒體帳戶或允許視頻下載，至少需要復制一個MP4。選擇沒有任何如果不應創建MP4再現。
  14. 選擇音頻翻譯應該創建的。
  15. 點擊保存保存新的配置文件。
請注意，如果您需要具有特定分辨率的再現，則可以在配置文件中要求，但不能在 Studio 中要求。相反地，您將需要使用內嵌設定檔 API 。
請注意，要求演繹是 不是 推薦，因為它會降低上下文感知編碼的效率。如果您有數個必要的解決方案，CAE 可能不適合您。

複製攝取配置文件

如果需要創建與現有配置文件相似的配置文件，則可以復制現有配置文件，然後對其進行編輯以進行所需的更改。

要復制攝取配置文件，請執行以下操作:

  1. 點擊行政 Studio標頭中的鏈接。
  2. 點擊攝取個人資料關聯。
  3. 單擊提取配置文件進行複制。
  4. 點擊複製。
  5. 使用概要文件編輯器來編輯概要文件名稱和定義。
  6. 點擊保存。

編輯攝取配置文件

攝取配置文件管理頁面可用於編輯攝取配置文件。請注意，默認的攝取配置文件集（已標記為標準視頻點播）無法編輯。

要編輯攝取配置文件，請執行以下操作:

  1. 點擊行政 Studio標頭中的鏈接。
  2. 點擊攝取個人資料關聯。
  3. 單擊攝取配置文件進行編輯。
  4. 使用概要文件編輯器來編輯概要文件定義。
  5. 點擊保存。

刪除攝取配置文件

“攝取配置文件"管理頁面可用於刪除不再需要的攝取配置文件。請注意，默認的攝取配置文件集（已標記為標準視頻點播）無法刪除，但他們可以被隱藏如果不需要它們。

要刪除提取的個人資料，請執行以下操作:

  1. 點擊行政 Studio標頭中的鏈接。
  2. 點擊攝取個人資料關聯。提取配置文件管理頁面將打開，顯示為您的帳戶創建的提取配置文件。
  3. 單擊刪除圖標 () 旁邊的攝取配置文件或單擊配置文件名稱，然後單擊刪除按鈕。
  4. 通過單擊確認刪除刪除個人資料。

更改上傳默認攝取配置文件

所有的Video Cloud帳戶都有一個上傳默認攝取配置文件。上傳的默認攝取配置文件用 在配置文件名稱旁邊。

要更改該帳戶的默認配置文件，請執行以下操作:

  1. 點擊行政 Studio標頭中的鏈接。
  2. 點擊攝取個人資料關聯。將打開“攝取配置文件"管理頁面，顯示為該帳戶創建的攝取配置文件。
  3. 單擊您想要設置為默認配置文件的攝取配置文件名稱。
  4. 點擊上載中旁邊默認使用頁面頂部的鏈接。

隱藏和顯示攝取配置文件

通過將與配置文件相關聯的滑塊單擊到隱藏位置。這對於隱藏可能不需要的默認配置文件很有用。

隱藏的攝取配置文件不會出現在“上載"模塊的“攝取配置文件"列表中。

配置文件中的DRM支持

聯繫Brightcove支持為視頻的DRM打包設置默認值。您無需在攝取配置文件中指定DRM包裝。如果您的帳戶啟用了DRM，並且您想交付一些沒有DRM的標題，請在“媒體"模塊中編輯視頻屬性，然後將視頻的DRM標誌設置為沒有DRM。

筆記: 如需有關建立已啟用 DRM 的 Brightcove 播放器的資訊，請參閱 DRM 外掛程式主題。

使用高級編輯器

接收配置文件是使用JSON定義的。要查看配置文件的JSON語法，請編輯配置文件，然後單擊進階編輯標籤。配置文件的JSON表示將顯示。

雖然可以在此處編輯JSON，但是Brightcove建議使用簡單編輯器編輯配置文件，以減少出錯的可能性。有關可用屬性的信息，請參見動態傳送和上下文感知編碼的攝取配置文件API字段參考話題。
