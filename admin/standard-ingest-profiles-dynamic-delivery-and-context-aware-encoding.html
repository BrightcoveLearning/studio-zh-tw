介紹

使用動態攝取 API或使用上傳模塊上傳視頻，您必須提供一個配置文件，用於確定視頻云攝取過程創建的再現的數量和類型。本主題提供有關每個動態投放和上下文感知編碼（CAE）個人資料，包括HEVC（h.265）個人資料。

筆記:
  • 上下文感知編碼和HEVC編碼是Video Cloud的付費附加組件。如果您有興趣將這些功能添加到您的帳戶，請與您的客戶經理聯繫。

有關動態投放的概述，請參見動態交付概述。

有關上下文感知編碼的概述，請參見上下文感知編碼概述。

有關HEVC的概述，請參見HEVC視頻

筆記:

  • 本文檔顯示了常規動態交付和上下文感知編碼的配置文件，包括HEVC配置文件
  • 由於靜態動態交付和上下文感知編碼配置文件的工作方式不同，因此您將在以下配置文件詳細信息中看到不同的信息:
    • 對於靜態動態交付配置文件，將顯示包含的演繹列表
    • 對於上下文感知編碼配置文件，顯示了一個動態配置文件設置表，該表用於確定要產生的演繹形式

也可以創建自己的自定義攝取配置文件。有關信息，請參見適當的主題:

  • 創建用於動態交付的提取配置文件
  • 創建用於上下文感知編碼的攝取配置文件

音頻再現

分段音頻再現

分段音頻渲染
名稱 [ 1-1] 音頻比特率（kbps） 音訊頻道 音頻編解碼器 強制關鍵幀速率 片段持續時間（毫秒） 片段軌道時標 段長度（秒）
筆記
  • [ 1-1] 當您在攝取配置文件中引用再現時，確保在名稱前加上default/ - 例如: default/audio128
  • [ 1-2] 段與可下載文件相關。例如，使用HLS，您可能每6秒對文件進行一次分段。每個片段包含足夠的信息，以使其能夠獨立於其他片段進行解碼和播放。HLS清單將列出每個段路徑及其持續時間。

    片段是指MP4文件的片段。無碎片的MP4在文件的開頭或結尾處包含元數據，並且要求數據（FTYP，MOOV原子）先於媒體（MDAT原子）。為了簡化流傳輸，可以對MP4文件進行分段，以便將元數據，播放所需的元數據以及媒體分段成文件中的片段。這使玩家可以輕鬆地搜索特定的媒體塊，並允許在不同演繹形式之間進行平滑過渡。碎片最初是為DASH創建的，但現在也由HLS支持。

    您還可以對媒體進行分段和破折號處理，以便一個段（文件）可以包含多個分段。通常使段和片段持續時間相同。注意duration和length討論片段和片段時指的是同一件事。

HEVC 視頻播放 [ 4-1]

HEVC 視頻再現 [ 4-4]
名稱 [ 4-3] 比特率（kbps） 高度 [ 4-2] (像素) 外觀模式 解碼器比特率上限 解碼器緩衝區大小 關鍵幀率 片段持續時間（毫秒） 速度 視頻編解碼器
筆記
  • [ 4-1] HEVC 支持是付費插件。如果您對HEVC支持感興趣，請與您的客戶經理聯繫
  • [ 4-2] 16:9 縱橫比也包括寬度，但輸出再現的實際寬度取決於輸入源縱橫比
  • [ 4-3] 當您在攝取配置文件中引用再現時，確保在名稱前加上default/ - 例如: default/video2000
  • [ 4-4] 除了表中顯示的那些之外，HEVC 再現還具有以下屬性:
    • fixed_keyframe_interval: true
    • segment_seconds: 2
    • video_bframes: 3
    • video_codec_profile: main
    • video_reference_frames: 6

名稱 [ 5-2] 視頻比特率（kbps） 音頻比特率（kbps） 高度 [ 5-1] (像素) 解碼器比特率上限 解碼器緩衝區大小 H264簡介
筆記
  • [ 5-0] 除了上面顯示的屬性之外，所有漸進式再現都具有以下設置:
    • 外觀模式: 保存
    • 速度: 3
    • 視頻編解碼器: h264
    • 高檔: 假
    • 音頻編解碼器: AAC
    • AAC配置文件: aac-lc
  • [ 5-1] 輸出再現的寬度取決於輸入源縱橫比
  • [ 5-2] 當您在攝取配置文件中引用再現時，確保在名稱前加上default/ - 例如: default/progressive1200
