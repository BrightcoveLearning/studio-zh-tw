定向遷移工具提供了一種指定應加載Brightcove Player而不是Smart Player的方法。該工具提供了一種在Brightcove Player內發布視頻的簡便方法，而不是使用傳統的Smart Player。無需更改現有的發布代碼。

本主題包含以下部分:

  • 定向遷移要求
  • 創建映射
  • 測試玩家映射
  • 編輯播放器映射
  • 刪除玩家映射
筆記: 智慧玩家到布萊特灣玩家對應將在 2017 年底的智慧玩家生命結束後繼續運作。

定向遷移要求

使用定向遷移工具時，需要滿足以下要求:

  • 僅當您的帳戶中有3個或更多Smart Player時，才會顯示Directed Migration UI
  • 您應該創建一個具有與Brightcove Player兼容的樣式，自定義項和插件的新Brightcove Player
  • 具有播放列表的播放器不支持定向遷移工具。
  • 更換播放器後，現有頁面中的所有Smart Player API代碼將不再起作用。

局限性

請注意定向遷移工具的以下限制:

  • 使用Smart Player嵌入發布代碼或使用簡短或自定義URL（bcove.me或link.brightcove.com）時，定向遷移將起作用。
  • 定向遷移不適用於Legacy Studio中的僅Flash嵌入代碼。

創建映射

映射用於指定將Smart Player替換為Brightcove Player。若要進行新的映射，請按照下列步驟操作:

  1. 點擊行政 Studio標頭中的鏈接。
  2. 點擊播放器設定關聯。
  3. 點擊創建映射。將顯示該帳戶中的智能播放器列表。
  4. 選擇一個或多個智能播放器以替換Brightcove播放器。點擊下一個。
  5. 選擇一個Brightcove播放器以代替智能播放器。選擇Brightcove播放器時，將顯示播放器的預覽。
  6. 點擊下一個。說明測試玩家映射將顯示。

    筆記: 播放器映射最多可能需要5分鐘才能生效。

  7. 點擊保存。將顯示當前映射的列表。

創建映射後，狀態為測驗。一旦映射已經過測試並承諾，狀態將是堅定的。標記為狀態的映射堅定的將渲染Brightcove Player而不是Smart Player。

測試玩家映射

您應該測試在提交之前創建的所有播放器映射。要測試玩家映射，請按照下列步驟操作:

  1. 查找顯示您的舊版Smart Player的頁面。
  2. 添加enableMapping=true頁面URL的參數。
    • 如果頁面URL沒有URL參數，請添加?enableMapping=true到URL的末尾。例如:
      http://www.mysite.com/videos.html?enableMapping=true
    • 如果頁面URL具有URL參數，請添加&enableMapping=true到URL的末尾。例如:
      http://www.mysite.com/videos.html?myparams=true&enableMapping=true

    將顯示新映射的Brightcove Player，並播放視頻。

    當您對播放器的工作方式感到滿意時，請返回Video Cloud Studio。

  3. 選中播放器映射旁邊的複選框，然後單擊提交映射使所有站點訪問者都可以看到新的Brightcove Player。

編輯播放器映射

要編輯玩家映射，請按照下列步驟操作:

  1. 點擊行政 Studio標頭中的鏈接。
  2. 點擊播放器設定關聯。將顯示玩家映射列表。
  3. 選中要編輯的播放器映射旁邊的複選框。點擊編輯映射。
  4. 選擇一個新的Brightcove Player進行映射。點擊下一個然後保存保存更改。

確保你測試對玩家映射的任何更改。

刪除玩家映射

要刪除播放器映射，請按照下列步驟操作:

  1. 點擊行政 Studio標頭中的鏈接。
  2. 點擊播放器設定關聯。將顯示玩家映射列表。
  3. 選中要刪除的播放器映射旁邊的複選框。點擊刪除映射。
  4. 通過單擊確認刪除刪除。
