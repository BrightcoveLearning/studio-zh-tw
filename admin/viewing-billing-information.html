您可以使用Video Cloud Studio查看帳戶的賬單信息。請注意，要訪問“結算"頁面，您必須是具有管理員角色的Video Cloud Studio用戶，並且您必須與您的帳戶有年度合同。要訪問“結算"頁面，請執行以下操作:

  1. 登錄到Video Cloud Studio。
  2. 在導航標題中，單擊行政然後開票。

將顯示“結算"主頁面。

筆記:
  • 如果一個帳單帳戶下拉列表顯示在頁面頂部，這表明您與Brightcove有多個合同。使用下拉菜單選擇合同以查看帳單明細。
  • 如果您擁有一個或多個Zencoder帳戶以及Video Cloud帳戶，則VOD Zencoder使用分鐘數項目可能會出現在Video Cloud帳單上。包括這些信息是為了方便您，只是為了讓您完整了解帳單-這些費用與Video Cloud無關。
  • 刪除視頻後，我們會將其保留在系統中12天，以防您需要恢復該決定。因此，刪除的視頻將在刪除後的12天內繼續反映在託管內容中。

帳單指標

“帳單"頁面上顯示的數據將根據您擁有的帳戶數量和帳戶權利而有所不同。例如，如果您不使用DRM，則不會顯示該信息。

筆記: 前一個月的帳單資訊在當月 7 日之前無法使用。

合同概述部分可能會顯示以下計費指標。

帶寬

帶寬度量標準由2個子度量標準組成，分別是視頻點播邊緣帶寬加居住邊緣帶寬。它顯示總帶寬/合同規定的帶寬。總帶寬包括視頻和播放器負載。該度量標準名稱的名稱將與您在計費發票中看到的名稱相匹配，以跟踪帶寬使用情況（即，它可能不會顯示為“帶寬"）。如果帶寬量超過配額，該圖形將顯示為紅色。條形圖可用於查看帶寬迄今為止的合同或者本月至今。

筆記: 帶寬數據最多延遲3-5天，儘管通常可以更快地獲得。數據部分取決於來自 CDN 的信息，因此可能會出現延遲，超出我們控制的範圍。

溪流

“數據流"度量標準顯示“總數據流開始" /“合同規定的數據流開始"。這應與Google Analytics（分析）模塊中的視頻觀看次數相匹配。如果流啟動的數量超過配額，則該圖形將顯示為紅色。條形圖可用於查看流（以千為單位）迄今為止的合同或者本月至今。

筆記: 串流開始數對應於 Analytics (分析) 模組中的視訊檢視次數。
筆記: 比較賬單數據和分析數據時，您可能會發現差異。這可能是由於將對帳後的結算數據與未對帳的分析數據進行了比較。調解程序需要 CDN 提供的伺服器記錄檔，而且取得伺服器記錄檔和處理資料時可能會有延遲。

加密許可證

“加密許可證"度量標準將顯示DRM許可證/合同約定的許可證總數。如果使用的許可證數量超出配額，則該圖形將顯示為紅色。條形圖可用於查看已使用的許可證數量迄今為止的合同或者本月至今。

請注意，無論播放器預加載設置如何，加密總是在視頻流式傳輸之前加載，以避免加載視頻的延遲。基於這個原因，您可能會在計費量度中看到比「串流」更多的「加密授權」。

貯存

存儲量指標將顯示總存儲量/合同規定的存儲量。如果使用的存儲空間超出配額，則該圖形將顯示為紅色。條形圖可用於查看已使用的存儲迄今為止的合同或者本月至今。

筆記:

  • 僅針對專業和企業帳戶顯示存儲。
  • 視頻演繹/資產和數字母帶分別存儲，但是“存儲"度量標準代表了演繹/資產和數字母帶的總合併存儲量。

轉碼

轉碼是指Brightcove與將視頻文件轉換為其他視頻格式或比特率相關的輸出GB總量。如果使用的轉碼超出配額，則該圖形將顯示為紅色。條形圖可用於查看所使用的轉碼迄今為止的合同或者本月至今。

原始服務帶寬

僅傳送的字節的帶寬使用情況（適用於BYO CDN發布者）。

投放的廣告

投放的廣告總數。這也與產品的整體價值有關。該指標還使用實時模塊為下游抵押品提供數據，或為發布者提供有關ROI的信息。

有效帶寬

輸入您的內容轉碼的小時數。

Brightcove實時輸出層

Brightcove實時輸出層度量標準將顯示總實時小時數/合同實時小時數。如果小時數超出配額，則該圖形將顯示為紅色。條形圖可用於查看已用小時數迄今為止的合同或者本月至今。選擇此度量標准後，每個層的輸出小時數將顯示在條形圖上方。

事件發生後，需要3天才能使用“實時"模塊小時數和“實時帶寬"出現在“結算用戶界面"中。按攝取等級計費如下:

  • 第1層的收費時間為1.0x小時
  • 第2層的收費時間為1.5倍
  • 第3層的收費時間為2.0x小時

使用時自定義實時攝取配置文件，則該層將取決於實時配置文件的總比特率。請注意，對於RTMP輸出，從作業開始到結束都將開始計費，包括關閉編碼器時的重新連接超時。

Brightcove Live DVR 營業時間

這Brightcove Live DVR 營業時間顯示轉碼的輸入秒數 + 啟用 DVR 的實時視頻。如果小時數超出配額，則該圖形將顯示為紅色。

活著的起源

實時起源帶寬被定義為CDN的高速緩存未命中以及實時起源需要滿足的請求。轉售帶寬時，這會對系統產生成本影響，並將有助於評估每位客戶的利潤和平台的整體盈利能力，從而推動未來的定價討論。

筆記: 「計費」模組中無法使用即時 Origin 資料。

VOD起源

實時VOD原始帶寬（BYO CDN）。

查看帳戶詳細信息

頁面的“帳戶詳細信息"部分將顯示所選指標的詳細信息。要選擇要查看的指標，請單擊“合同概述"部分中的一項，或使用“帳戶詳細信息"部分中的下拉列表選擇一項。

條形圖將更新以顯示所選指標的詳細信息。將顯示當月和過去12個月的數據。將鼠標懸停在圖表上會顯示該月的詳細信息。

但是，如果將鼠標懸停在當月，請注意，這是不是實時數據。當前月份的數據僅是當前日期之前四天的最新數據，並且僅是估計值。

條形圖下方將顯示所有帳戶的列表。默認情況下，如果您有多個帳戶，將顯示所有帳戶的數據。要查看單個帳戶的數據，請通過單擊該帳戶的單選按鈕或單擊該行來選擇一個帳戶。

匯出資料

去下載開具發票 CSV 格式的賬單數據，點擊CSV在頁眉中。

刪除的視頻和數字母帶

請注意，刪除視頻，數字母帶和其他資產時，不會立即刪除物理文件。我們將它們保留到位至少7天（有時長達14天），其原因有兩個:

  • 如果它們被誤刪除而您想恢復它們
  • 這樣您仍然可以獲取視頻的元數據和分析數據

只要物理文件仍然存在，您就可以將要查看帳單中反映的已刪除媒體資產。

賬戶鎖定

在長達60天的時間裡，鎖定帳戶的存儲將繼續反映在“結算"數據中。
