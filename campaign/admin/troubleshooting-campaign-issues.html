啟用詳細日誌記錄

啟用詳細日誌記錄允許您在瀏覽器控制台中查看與營銷活動相關的信息。這在嘗試調試 Campaign 問題時很有幫助。可以通過添加啟用詳細日誌記錄 bcverbose=1到啟用了戰役的播放器的頁面的 URL 或通過配置戰役插件"verbose": true選項:

日誌消息可以在控制台中查看。

潛在客戶表格問題

以下是我們在潛在客戶表格中遇到的一些常見問題和疑問。

為什麼即使提交表單後，潛在客戶表單仍繼續顯示Marketo Munchkin集成？

Munchkin集成沒有可用來詢問用戶是否已知的API。由於我們不知道他們是否是已知用戶，因此我們始終顯示銷售線索表格。Marketo REST集成確實提供了API，因此我們可以確定已知用戶與未知用戶。建議切換到Marketo REST集成，這樣可以避免此問題。

我的自定義潛在客戶表單設置為在提交時重定向。提交表單後，為什麼我看到頁面被重定向到一兩秒鐘？

我們必須等到重定向到的頁面已完全加載後，才能關閉潛在客戶表單。一種解決方案是將頁面重定向到輕量級頁面（沒有很多圖像或JavaScript），或者重定向到我們設置的空白頁面（位於https://map.brightcove.com/lead/complete）專門解決此問題。

我用確認HTML配置了潛在客戶表單。為什麼提交後幾秒鐘我會看到黑屏？

確保更改HTML的文本顏色。確認HTML可能是可見的，但您看不到它，因為默認文本顏色是黑色。

提交後，為什麼我的HubSpot潛在客戶表格不消失？

確保將HubSpot中的潛在客戶表單配置為重定向到http://map.brightcove.com/lead/complete。如果要顯示確認消息，請在 Campaign 而不是 HubSpot 中進行配置。

為什麼我的潛在客戶表單不顯示？

如果您的潛在客戶表單未顯示，請首先確保已滿足所有顯示潛在客戶表單的條件。請注意，取決於正在觀看的多個視頻的條件在整個頁面刷新中均不起作用。它們用於以下情況: 下一個視頻自動在同一播放器中排隊。

  1. 確認已將 Campaign 插件添加到播放器中。有關信息，請參閱創建支持戰役的播放器 .
  2. 確認已正確設置集成並將其發送到插件。
  3. 確認線索表已添加到該玩家。如果已將線索表添加到播放器，則插件對其配置進行的調用的響應應在leadForm屬性下包含線索表顯示信息。
  4. 確認查看者還不是潛在客戶。僅當 Campaign 確定查看者還不是營銷自動化平台 (MAP) 中的潛在客戶或尚未填寫潛在客戶表單時，才會顯示潛在客戶表單。該插件對 Campaign 服務器進行單獨的 JSONP 調用，以檢查查看者是否是潛在客戶。如果查看者是潛在顧客，則服務器將以狀態1和任何其他標識符進行響應，這對於將視圖數據同步到MAP可能很有用。如果查看者不是線索，則服務器將以狀態0進行響應。
    • 對於不是潛在客戶的觀看者的示例響應: 
      /**/ 回調的類型 = = '函數' 和回調函{"status":0}數;
    • 對於潛在客戶的示例響應: 
      /**/ 回調的類型 = = '函數' 和回調函數; {"status":1,"eloqua":{"contactId":"5","internalId":"CTCPB000000000005"}

同步問題

以下是一些常見的同步問題。

Marketo REST視圖事件無法與錯誤活動類型ID'000000'找不到的同步

與集成關聯的自定義Marketo活動已刪除時，會發生這種情況。創建Marketo REST集成後，我們將獲取活動，存儲其ID並使用該ID來同步事件。解決方案（假設已創建新的Brightcove Video View活動）是斷開連接並重新創建集成以獲取新活動的ID。

查看未同步的事件

使用 MAP 正確設置集成後，查看來自支持戰役的播放器的數據應填充所有最近的活動與相應 MAP 中的已知潛在客戶對應的頁面和查看事件最終應同步到該工具。

首先，請確保視圖數據顯示在最近所有活動頁:

如果查看事件已列出但不在 MAP 中，則用戶可能已將其 MAP 授權給 Campaign 作為沒有足夠權限通過 API 將查看數據導入 MAP 的用戶，或者未正確配置集成。活動主頁列出了最近查看事件同步操作的狀態。那裡應該有反饋，以指示為什麼視圖事件未同步到MAP。

如果視圖事件顯示在 Campaign 中，請檢查地位列是已同步 .這表明Campaign 能夠成功地將數據傳送到您的MAP 中。如果您看到狀態為unknown_contact，這表示該用戶未知，而您的MAP無法接收該用戶。大多數MAP僅接受已知用戶的第三方數據。解決方案是使用在MAP中創建的自定義潛在客戶表單，以確保您確實是已知用戶。

如果未列出查看事件，則可能尚未使用支持戰役的播放器設置戰役。使用啟用了開發人員工具的瀏覽器，訪問包含有問題視頻的頁面，並確保 Campaign 插件進行某些 HTTP 調用。

當播放器加載時，插件應調用 Campaign 配置端點以獲取根據集成確定插件行為的參數:

如果網絡通信中缺少此呼叫，則表示尚未為播放器分配集成。有關如何執行此操作的信息，請參閱創建支持戰役的播放器 .

該調用是使用JSONP進行的。如果JSONP響應是一個空對象，則可能是嵌入代碼不正確並指定了錯誤的帳戶ID。如果配置請求未能同時包含帳戶和玩家 URL 參數，或者無法從特定於戰役的數據庫中檢索到指定的帳戶（很可能在 URL/嵌入中指定了錯誤的帳戶），則此端點將返回一個空對象代碼，但在數據庫中未找到）。

/**/ 關鍵字回調的類型 = = '函數' {}

如果對象具有ID參數，但缺少集成參數，則用戶尚未與其營銷自動化工具建立集成，並且不會記錄查看事件。

如果在發布支持戰役的播放器時使用標準嵌入 (iframe) 代碼，您也會遇到數據同步問題。發布播放器時，請使用“高級嵌入"（頁內）代碼。標準嵌入 (iframe) 發布代碼將不起作用，因為 Campaign 插件將在 iframe 內運行，因此將無法訪問父頁面 cookie 或 網址參數。

連接問題

Campaign 模塊說它無法設置 Eloqua 集成

檢查Eloqua的要求。如果您使用 IP 白名單安全功能，Campaign 將無法與 Eloqua 通信，除非您將指定的 IP 地址列入白名單以允許 Campaign - Eloqua 流量。接觸Brightcove支持想要查詢更多的信息。

Campaign 說我的 Marketo REST 自定義活動缺少字段，但是當我查看 Marketo 時，我可以看到所有字段都存在

有幾件事要檢查:

  1. 字段名稱必須完全匹配。“事件1"與“事件1"不同。
  2. 活動必須獲得批准，並且處於草稿狀態。
  3. 如果刪除了自定義活動並在沒有設置新的Marketo REST集成的情況下重新創建了自定義活動，則必須斷開舊的集成並創建一個新的集成。我們查找自定義活動，並為每個集成存儲其ID值。如果通過刪除ID進行更改，我們將無法自動提取新ID。
  4. 如果最近在活動中添加了新字段，則更改最多可能需要24小時才能在Marketo的系統中生效。
