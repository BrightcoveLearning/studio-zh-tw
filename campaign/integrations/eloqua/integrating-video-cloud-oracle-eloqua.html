Campaign 模塊允許 Video Cloud 將觀看數據從 Brightcove Players 同步到 Oracle Eloqua。進入Oracle Eloqua後，該數據可用於細分，活動邏輯，報告，潛在客戶評分和個性化通信。Campaign 模塊與 Oracle Eloqua 的集成包括兩個主要組件:

  • Brightcove Player插件，用於跟踪視頻觀看數據
  • 從Video Cloud向Oracle Eloqua發送跟踪數據的同步過程

要求

要將 Campaign 模塊與 Oracle Eloqua 結合使用，您必須:

  • 擁有專業或企業視頻雲帳戶
  • 通過在打開活動模塊時出現的初始啟動屏幕中提交所需信息，為您的帳戶啟用活動模塊
  • 首次使用該模塊時，請單擊服務協議
  • 發布播放器時，請使用高級嵌入（頁內）發布代碼；由於該插件將在iframe中運行，因此標準嵌入（iframe）發布代碼將無法正常工作，因此將無法訪問任何父頁面URL參數，並且可能無法獲得正確的Cookie
  • 使用 Oracle Eloqua 10 和憑據登錄: https://login.eloqua.com
    筆記: 這些憑證會輸入 Oracle Eloqua 登入頁面，而且不會直接提供給布萊特灣。
  • 用於登錄Oracle Eloqua的證書應添加到高級用戶-營銷組並具有以下權限:
    • 消耗API
    • 安裝雲應用
    • 設置營銷活動

特徵

視頻雲-Oracle Eloqua集成的一些關鍵功能包括:

  • 潛在客戶表格 -創建Eloqua表單，這些表單會在觀眾觀看視頻時自動顯示。有關信息，請參閱為Oracle Eloqua創建自定義潛在客戶表單。
  • 資料格式 -視頻雲查看數據被傳遞到Oracle Eloqua中的自定義數據對象（CDO）。複製到Oracle Eloqua的數據如下:
    • 影片名稱
    • 影片編號
    • 帳戶ID（視頻雲）
    • 頁面URL（引薦頁面的URL）
    • 播放器ID（視頻雲播放器）
    • 觀看百分比
  • 數據傳輸 -查看數據每小時同步一次，可能需要2-4個小時才能出現在Oracle Eloqua中。
  • 資料保留 -收集的數據可以選擇保留在Video Cloud中長達1個月。

設置連接

在配置與Eloqua的集成之前，我們建議您打開一個新的瀏覽器窗口並手動註銷您的Eloqua帳戶。這將確保在配置集成時，將提示您登錄Eloqua帳戶，並且可以確認您使用的帳戶正確。

請按照以下步驟在Video Cloud和Oracle Eloqua之間建立連接。

  1. 打開活動模塊並點擊數據連接在左側導航中。
  2. 點擊添加新連接。
  3. 選擇埃洛誇作為整合。
    筆記: 可用的資料連線是根據您的 Video Cloud 訂閱而定。
  4. （可選）輸入連接名稱。
  5. 點擊連接。
  6. 點擊連接到Eloqua。
  7. 系統將提示您登錄Eloqua。輸入您的Eloqua 公司名稱，用戶名和密碼。點擊登入。
  8. 如果登錄成功，則將提示您安裝Brightcove Video Connect for Eloqua應用程序。點擊接受並安裝。
  9. 系統將提示您啟用該帳戶的Eloqua。點擊使能夠。
  10. 系統將提示您允許Brightcove Video Connect for Eloqua訪問您的Eloqua帳戶。點擊登入。
  11. 確認您的公司名稱和用戶名，然後單擊接受。
  12. 系統將提示您設置Eloqua。點擊設置Eloqua。

配置集成

一旦建立了Video Cloud與Oracle Eloqua的連接，就可以配置集成。

  1. 查看將視頻觀看同步到Eloqua作為外部活動將視圖作為外部活動進行同步。外部活動是在Eloqua中提供信息的另一個地方。發布者可以選擇使用外部活動在Eloqua中進行報告或觸發操作。有關更多信息，請參見Eloqua。文件資料。
  2. 默認情況下，連接狀態將已啟用。這連接狀態確定是否將Video Cloud視頻參與度數據發送到Oracle Eloqua。
    • 已啟用 -收集數據並將其發送到Eloqua
    • 殘障人士 -未收集數據（禁用集成）
  3. 默認，跟踪未知觀看者的數據將殘障人士。此設置確定當觀看者未知時是否將視頻參與度數據保存在視頻雲中。僅將已知觀眾的數據發送到Eloqua。
    • 已啟用 -收集所有觀眾的數據並將其保存在Video Cloud中
    • 殘障人士 -未知觀眾的數據不會保存在Video Cloud中
  4. 選擇數據保留策略。數據保留政策決定了觀看數據和潛在客戶表單數據（如果使用廣告系列潛在客戶表單）在您的 Video Cloud 帳戶中保留多長時間。將視頻保留在Video Cloud中後，您可以從中斷的同步中恢復。此外，Campaign 將繼續重試無法同步的事件（通常是因為觀看者在觀看視頻時不是已知聯繫人），直到事件超過數據保留期。您應根據您的隱私政策調整保留設置 -不需要在Video Cloud中保留任何用戶數據。數據保留期過後，數據將被刪除。

要刪除與Oracle Eloqua的集成，請單擊斷開。頁面底部將列出所有啟用戰役的玩家和相關的潛在客戶表單。

Campaign 模塊支持與營銷平台的多個連接。點擊添加新連接配置其他集成。

驗證整合

配置集成後，請測試集成以確認視頻觀看數據已發送到Eloqua。有關驗證集成的信息，請參見Eloqua集成測試指南。

創建支持戰役的播放器

僅當您使用支持 Campaign 的視頻播放器時，視頻參與度數據才會發送到 Eloqua。有關創建支持戰役的 Brightcove 播放器的信息，請參閱創建支持戰役的播放器 .

傳遞跟踪ID

Campaign 模塊支持將跟踪 ID 值傳遞給 Eloqua 的功能。例如，跟踪ID可能用於跟踪廣告系列ID。跟踪 ID 可以附加到顯示已啟用戰役的播放器的 URL 或添加到播放器嵌入代碼。

  • 例子: 附加到URL
  http://www.myurl.com/video.html？ bctrackingid = 12345
  • 例子: 添加到播放器嵌入代碼
  <video data-video-id="6033402539001" data-account="1486906377" data-player="default" data-embed="default" data-application-id class="video-js" 數據 bc-trackingid="12345"控制></視頻><腳本 src="//players.brightcove.net/1486906377/default_default/index.min.js">< /腳本>

然後，Campaign 會將值作為另一個同步字段傳遞給 Eloqua。

識別觀眾

具有 Eloqua 集成的支持戰役的玩家將調用 Eloqua 的 API 來獲取訪問者 ID，通常稱為 Eloqua GUID。此 ID 在 Campaign 模塊的查看數據中捕獲，用於將查看數據同步到 Eloqua。

Eloqua 使用自己的一組 cookie，當 Brightcove 播放器配置了 Eloqua 集成時，Campaign 會在 Eloqua cookie 中捕獲 GUID 並與 Eloqua 服務器端通信，以嘗試找出用戶是誰。

用戶的電子郵件地址也可以傳遞並用於標識查看者。有關更多信息和示例，請參閱活動模塊如何識別查看者 .
