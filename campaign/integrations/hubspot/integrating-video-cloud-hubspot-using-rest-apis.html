Campaign 模塊允許 Video Cloud 將觀看數據從 Brightcove 播放器同步到 HubSpot。進入HubSpot後，此數據可用於細分，廣告系列邏輯，報告，潛在客戶評分和個性化通信。

有兩種方法可以將Video Cloud與HubSpot集成:

  • 使用標準 (REST API) 方法 - 使用此方法，Campaign 將僅跟踪已知觀眾的觀看數據。默認情況下，潛在客戶表單只會顯示給未知的查看者。本主題涵蓋了此方法。
  • 使用高容量（客戶端API) 方法 - 使用此方法，Campaign 將跟踪所有觀眾的觀看數據。有關此類集成的信息，請參見使用客戶端API將視頻雲與HubSpot集成。

與 HubSpot 的 Campaign 模塊集成包括兩個主要組件:

  • Brightcove Player插件，用於跟踪視頻觀看數據
  • 從視頻雲向HubSpot發送跟踪數據的同步過程

要求

要將 Campaign 模塊與 HubSpot 結合使用，您必須:

  • 擁有一個HubSpot帳戶
  • 擁有專業或企業視頻雲帳戶
  • 通過在打開活動模塊時出現的初始啟動屏幕中提交所需信息，為您的帳戶啟用活動模塊
  • 首次使用該模塊時，請單擊服務協議
  • 發布播放器時，請使用高級嵌入（頁內）發布代碼；由於該插件將在iframe中運行，因此標準嵌入（iframe）發布代碼將無法正常工作，因此將無法訪問任何父頁面URL參數，並且可能無法獲得正確的Cookie

特徵

視頻雲-HubSpot集成的一些關鍵功能包括:

  • 潛在客戶表格 -創建HubSpot表單，這些表單會在觀眾觀看視頻時自動顯示。有關信息，請參閱為HubSpot創建自定義潛在客戶表單。
  • 資料格式 -查看數據是作為自定義提供的時間軸事件類型。複製到HubSpot的數據如下:
    • 影片名稱
    • 影片編號
    • 帳戶ID（視頻雲）
    • 頁面URL（引薦頁面的URL）
    • 播放器ID（視頻雲播放器）
    • 觀看百分比（25％，50％，75％，95％）
  • 數據傳輸 -使用自動過程將查看數據發送到HubSpot；但是，HubSpot平台中的某些報告可能需要一些時間才能刷新
  • 資料保留 -收集的數據可以選擇保留在Video Cloud中長達6個月

設置連接

請按照以下步驟在Video Cloud和HubSpot之間建立連接。

在Video Cloud和HubSpot之間建立連接

  1. 打開活動模塊並點擊數據連接在左側導航中。
  2. 點擊添加新連接。
  3. 選擇HubSpot作為整合。
    筆記: 可用的整合是根據您的 Video Cloud 訂閱而定。
  4. （可選）輸入連接名稱。
  5. 點擊連接。
  6. 選擇標準（REST API）然後點擊保存。
  7. 點擊連接到HubSpot。
  8. 系統將提示您登錄HubSpot。輸入您的電子郵件地址和密碼。點擊登錄。
  9. 如果登錄成功，系統會提示您選擇一個帳戶。點擊選擇賬戶 .

至HubSpot的視頻雲連接現已建立並可以使用。

配置集成

建立Video Cloud至HubSpot的連接後，即可配置集成。點擊HubSpot帳戶信息顯示連接的HubSpot帳戶信息。

  1. 默認情況下，連接狀態將已啟用。這連接狀態確定是否將Video Cloud視頻參與度數據發送到HubSpot。
    • 已啟用 -收集數據並將其發送到HubSpot
    • 殘障人士 -未收集數據（禁用集成）
  2. 默認，跟踪未知觀看者的數據將設置為殘障人士。啟用此選項將跟踪未知用戶的數據。數據將根據所選的數據保留策略保留。如果數據保留設置為沒有保留，則不會顯示此字段。
  3. 選擇數據保留策略。數據保留政策決定了觀看數據和潛在客戶表單數據（如果使用廣告系列潛在客戶表單）在您的 Video Cloud 帳戶中保留多長時間。將視頻保留在Video Cloud中後，您可以從中斷的同步中恢復。此外，Campaign 將繼續重試無法同步的事件（通常是因為觀看者在觀看視頻時不是已知聯繫人），直到事件超過數據保留期。您應根據您的隱私政策調整保留設置 -不需要在Video Cloud中保留任何用戶數據。數據保留期過後，數據將被刪除。

要刪除與HubSpot的集成，請單擊斷開。Campaign 模塊支持與營銷平台的多個連接。點擊添加新連接配置其他集成。

驗證整合

配置集成後，請測試集成以確認視頻觀看數據已發送到HubSpot。有關驗證集成的信息，請參見HubSpot REST API集成測試指南。

創建支持戰役的播放器

僅當您使用支持營銷活動的視頻播放器時，視頻參與度數據才會發送到 HubSpot。有關創建支持戰役的 Brightcove 播放器的信息，請參閱創建支持戰役的播放器 .

傳遞跟踪ID

Campaign 模塊支持將跟踪 ID 值傳遞給 HubSpot 的功能。例如，跟踪ID可能用於跟踪廣告系列ID。跟踪 ID 可以附加到顯示已啟用戰役的播放器的 URL 或添加到播放器嵌入代碼。

  • 例子: 附加到URL
  http://www.myurl.com/video.html？ bctrackingid = 12345
  • 例子: 添加到播放器嵌入代碼
  <video data-video-id="6033402539001" data-account="1486906377" data-player="default" data-embed="default" data-application-id class="video-js" 數據 bc-trackingid="12345"控制></視頻><腳本 src="//players.brightcove.net/1486906377/default_default/index.min.js">< /腳本>

然後，Campaign 會將值作為另一個同步字段傳遞給 HubSpot。

識別觀眾

為了進行觀看者跟踪，可以通過以下兩種方式之一來跟踪觀看者:

  • URL參數 -用戶的電子郵件地址可以通過郵件顯示視頻的頁面上的URL參數。有關更多信息和示例，請參閱活動模塊如何識別查看者 .
  • HubSpot cookie - 當播放器加載時，Campaign 會查看用戶是否已經被 HubSpot cookie。如果是，Campaign 將詢問 HubSpot 是否通過 cookie 值知道用戶是誰。如果 HubSpot 返回聯繫人，Campaign 會將所有查看數據與該用戶關聯。

一種可能的情況是使用為跟踪目的而唯一生成的URL創建視頻登錄頁面。然後，Campaign 模塊將使用 URL 參數來查找用戶服務器端並同步查看數據。

為了使播放器/插件讀取URL參數，應使用“高級嵌入"（頁內）發布代碼。由於該插件將在iframe中運行，因此標準嵌入（iframe）發布代碼將無法正常工作，因此無法訪問具有該參數的父頁面URL。
