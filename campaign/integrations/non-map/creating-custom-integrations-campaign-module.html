Campaign 模塊允許 Video Cloud 同步來自支持 Campaign 的觀看數據  Brightcove Players 到流行的營銷自動化平台。CampaignCampaign 模塊還可以使用自定義集成進行配置。通過自定義集成，查看數據將本地存儲在Video Cloud中，並且Audience API用於以編程方式提取數據。

Campaign 模塊集成包含兩個主要組件:

  • Brightcove Player插件，用於跟踪視頻觀看數據
  • 使用Audience API檢索觀看數據的自定義開發

要求

要使用 Campaign 模塊，您必須:

  • 擁有專業或企業視頻雲帳戶
  • 通過在打開活動模塊時出現的初始啟動屏幕中提交所需信息，為您的帳戶啟用活動模塊
  • 首次使用該模塊時，請單擊服務協議
  • 為了收集潛在客戶，潛在客戶表單必須配置並分配給玩家
  • 發布播放器時，請使用“高級（頁內）"嵌入代碼；標準（iframe）嵌入代碼將無法使用，因為該插件將在iframe中運行，因此將無法訪問父頁面的網址參數

設置連接

請按照以下步驟配置自定義集成。

  1. 打開活動模塊並點擊數據連接在左側導航中。
  2. 點擊添加新連接。
  3. 選擇風俗作為整合。
    筆記: 可用的整合是根據您的 Video Cloud 訂閱而定。
  4. （可選）輸入連接名稱。
  5. 點擊連接。

現在，自定義連接已建立並可以使用。

配置集成

建立連接後，您可以配置集成。

  1. 默認情況下，連接狀態將已啟用。這連接狀態確定是否保存了Video Cloud視頻參與度數據。
    • 已啟用 -收集數據並將其保存
    • 殘障人士 -未收集數據（禁用集成）
  2. 選擇數據保留策略。數據保留政策決定了觀看數據和潛在客戶表單數據（如果使用廣告系列潛在客戶表單）在您的 Video Cloud 帳戶中保留多長時間。
    筆記: 如果選擇沒有保留，將不會保存任何數據。當您使用 API 下載數據時，將不會有任何數據。

創建支持戰役的播放器

視頻參與數據僅在以下情況下才會保存 您使用的是支持廣告系列的視頻播放器。有關創建支持戰役的 Brightcove 播放器的信息，請參閱創建支持戰役的播放器 .

傳遞跟踪ID

Campaign 模塊支持傳遞跟踪 ID 的功能。例如，跟踪ID可能用於跟踪廣告系列ID。跟踪 ID 可以附加到顯示已啟用戰役的播放器的 URL 或添加到播放器嵌入代碼。

  • 例子: 附加到URL
  http://www.myurl.com/video.html？ bctrackingid = 12345
  • 例子: 添加到播放器嵌入代碼
<video data-video-id="6033402539001" data-account="1486906377" data-player="default" data-embed="default" data-application-id class="video-js" 數據 bc-trackingid="12345"控制></視頻><腳本 src="//players.brightcove.net/1486906377/default_default/index.min.js">< /腳本>

識別觀眾

使用自定義集成時，您有責任識別查看者。Campaign 插件需要兩個關鍵信息:

描述 URL參數 嵌入參數 價值
觀眾是否已知（需要潛在客戶形式） bcknownuser data-bc-known-user 0或者1
什麼是GUID（查看事件跟踪所需） bccustomguid data-bc-custom-guid string
 

這些值可以通過URL參數或播放器嵌入代碼傳遞。為了進行觀看者跟踪，應該在顯示視頻的頁面上將用戶ID作為URL參數傳遞。潛在客戶表單需要已知用戶參數（如果您希望潛在客戶查看者有條件地顯示潛在客戶表單）。

筆記: 為了使播放器/插件讀取URL參數，應使用高級（頁內）嵌入代碼。標準（iframe）嵌入代碼將無法工作，因為插件將在 iframe 內部運行，因此無法訪問具有該參數的父頁面 URL。

查看視頻數據

使用自定義集成時，有幾個選項可用於查看視頻數據:

  • 觀眾活動（事件） - 作為活動模塊的一部分提供，此選項可用於查看活動模塊內的視頻數據。有關信息，請參閱在戰役模塊中查看玩家事件 .
  • 導出數據 - 作為活動模塊的一部分提供，此選項可用於導出視頻視圖和潛在客戶數據。有關信息，請參閱從活動模塊導出活動數據 .
  • 受眾群體API -Audience API可用於以編程方式檢索視頻觀看和潛在客戶數據。有關信息，請參閱概述: 受眾群體API。
