介紹

Brightcove Cloud Playout是一個Studio模塊，通過它可以對VOD目錄中的項目進行計劃的播出，並與實時供稿結合，並將其作為線性通道進行流傳輸，同時具有Brightcove Live的可靠性和可擴展性。

Cloud Playout為計劃的編程頻道提供了強大的端到端解決方案，其中包括:

  • 24x7線性通道
  • 彈出/虛擬頻道
  • 模擬直播

Cloud Playout可以利用類似預錄內容的廣播式直播電視體驗，幫助您增加觀眾參與度並創造新的收入機會。使用現有的VOD資產創建預定的直播頻道。

特徵

Cloud Playout的功能包括:

  • 從視頻雲媒體庫導入VOD資產。
  • 在VOD資產和預定的實時供稿之間切換。
  • 實時，輕鬆，快速，可靠地對最後一分鐘的計劃變更做出反應。
  • 通過服務器端廣告插入（SSAI）來利用您的頻道內容獲利。
  • 配置您的頻道以循環您的程序。
  • 通過設備優化播放。
  • 即使在運行狀態下，也可以隨時通過拖放來重新安排程序時間表。
  • 通過Brightcove Live，Amazon S3存儲桶或通過RTMP輸出將第三方推送到您擁有和經營的財產。
  • 檢索頻道的EPG（電子節目指南）的API

局限性

適用以下假設和限制。

  • 您的視頻必須使用動態投放配置文件提取或重新轉碼，必須具有MP4格式
  • Cloud Playout支持的最大輸入比特率和分辨率為20MBPS / 1080p。這適用於VOD內容和實時輸入。
  • Brightcove建議使用至少2分鐘的視頻，以確保視頻和電子節目指南準確性之間的正確過渡
  • 當前添加到視頻的字幕（文本軌道）將不會顯示在Cloud Playout頻道中。字幕支持將在以後的版本中添加。
  • 直播活動的最小長度為5分鐘。
  • Cloud Playout中的每個直播事件都與Video Cloud遠程資產共享1: 1關係。該遠程資產無法在其他渠道中重複使用。如果直播事件已從頻道中刪除，則關聯的遠程資產將被自動刪除。刪除頻道後，所有關聯的遠程資產也將被自動刪除。
  • 影片共享當前無法從另一個帳戶添加到該帳戶中。
  • 當前正在播放直播事件，但沒有活動的供稿時，板岩內容將循環播放，直到直播供稿恢復或事件結束為止。在平板MP4開始播放之前，大約會有30秒的延遲。
  • 如果將視頻雲帳戶配置為地理限制，則該頻道輸出區域應該在與Video Cloud地理限制政策兼容的區域中創建。
  • 為了獲得最佳體驗，Brightcove建議使用Google Chrome瀏覽器。
