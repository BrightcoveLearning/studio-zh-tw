生成Cloud Playout頻道的嵌入代碼

要為Cloud Playout頻道生成嵌入代碼，您將使用實時模塊。有關使用“實時"模塊的完整詳細信息，請參見使用實時模塊創建和管理實時事件。

有兩種類型的嵌入代碼:

  • 標準 -iframe嵌入代碼。推薦用於大多數情況。此類型將播放器放置在<iframe>標記內。
  • 先進的 -頁內嵌入代碼。允許播放器直接存在於HTML頁面中，而不存在於iframe中。這為開發人員提供了易於訪問播放器以及關聯的屬性和事件的好處。

您還可以獲取已發布播放器的URL。這對於在瀏覽器中查看事件很有用。

要為現場活動生成嵌入代碼，請按照以下步驟操作:

  1. 打開居住模塊。
  2. 找到該頻道將要流式傳輸的現場活動。通常，該事件將位於支持標籤，並命名為渠道除非您選擇了要在創建頻道時流式傳輸的特定事件。
  3. 點擊事件名稱以打開控制室頁。
  4. 點擊發布和嵌入>播放器配置在左側導航欄中。
    筆記: 如果將頻道配置為啟用服務器端廣告（SSAI），則Brightcove Player版本6.18.3及更高版本必須使用。只有 6.18.3 版及以上的玩家才會出現在下拉列表中。
  5. 點擊播放器網址在瀏覽器中預覽事件或複制要使用的嵌入代碼。

如果程序是使用廣告插入選項設置為上，發布選項將包括選擇廣告配置下拉菜單中可以選擇要使用的廣告配置。
