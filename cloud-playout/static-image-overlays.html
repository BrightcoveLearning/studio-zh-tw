介紹

您可以使用靜態圖像疊加功能將靜態圖像疊加到 Cloud Playout 頻道中的視頻上。靜態圖像是沒有運動的靜止圖像。您準備圖像並將其存儲在 Cloud Playout 之外，並作為可公開訪問的、未經身份驗證的 http/https URL 提供給我們。

例子

  • 您希望在通道運行期間的特定時間插入靜態圖像疊加並運行 10 秒。您希望圖像疊加顯示在視頻幀的右下角。您希望圖像疊加具有 50% 的不透明度。
  • 您想要插入兩個靜態圖像疊加層，以便它們同時出現在視頻幀中，或者有一些重疊。您希望圖像疊加的顯示稍微重疊，以便一個圖像疊加出現在某個位置，並且當該圖像疊加仍在顯示時，另一個圖像疊加出現在另一個位置。

執行

  • 您疊加在視頻上的圖像應該是.tga或 .png 文件。
  • 使用新的側邊菜單項為帳戶創建和管理靜態疊加圖像:
  • 然後可以將疊加圖像與通道相關聯:

  • 您最多可以插入兩個層一次。每個圖像應該是一個分離層 .如果不同的鏡像在同一時間被安排在同一層上，或者如果它們在任何時間窗口內重疊，那麼稍後提供的鏡像將刪除同一層上的現有鏡像。
    • 層按 Z 順序排列，這意味著具有較高層值的疊加層被插入到具有較低層值的疊加層之上 .
  • 您可以將疊加設置為同時（或不）出現在底層視頻上，並且您可以將它們設置為彼此物理重疊（或不重疊）。
  • 您可以使用開始時間和持續時間配置每個圖像疊加:
    • 如果設置了持續時間，則最長持續時間為 24 小時
    • 如果未設置持續時間，覆蓋將繼續顯示直到停用
    • 開始時間必須是未來時間
  • 您可以配置 0 - 100 的不透明度。 0是透明的100完全不透明。
  • 您可以按照 x/y 坐標的指定將圖像疊加放置在視頻幀上。默認情況下，X、Y 坐標被視為 0, 0 ( 視頻幀的左上角 ):
    • 位置 X: 疊加左邊緣相對於視頻幀左邊緣的位置，以像素為單位。如果放置導致疊加層超出底層視頻的右邊緣，則疊加層將在右側裁剪。
    • 位置 Y : 疊加頂部邊緣相對於視頻幀頂部邊緣的位置，以像素為單位。如果放置導致疊加層超出底層視頻的底部邊緣，則疊加層將在底部被裁剪。
  • 漿紗: 這使用原件選項可用於分辨率以使用所提供圖像的原始分辨率:
    • 您可以提供自定義寬度和高度對於讓系統在將圖像應用於源視頻之前縮放圖像的分辨率:
    • 離開寬度和高度空和使用原件未選中將導致系統使用通道的輸出分辨率。例如: 移動和 Web 輸出會將覆蓋分辨率設置為 720p，而智能電視配置文件會將覆蓋分辨率設置為 1080p。不推薦使用此選項因為它可能並不總是與 MP4 源分辨率對齊。

狀態轉換

覆蓋計劃的狀態轉換

  • 不活動
    • 新創建的時間表（或）
    • 已完成運行（或）的計劃
    • 已停用的計劃。
    • 您可以更改開始時間以使用相同或更新的覆蓋配置將其恢復到 SCHEDULED 狀態。
    • 刪除 - 如果您將不再使用此配置。
    • 修改任何配置更改。
  • 預定
    • 計劃在指定的開始時間開始。
    • 如果您不再需要此時間表，請刪除。
    • 修改任何配置更改。
  • 積極的
    • 當前正在進行的計劃。
    • 當您需要停止覆蓋時停用。
    • 修改除 layer 和 start_time 之外的任何配置。
    • 不允許刪除。您需要先停用。
您可以隨時編輯疊加層。然而層和開始時間不能更改為積極的覆蓋。您仍然可以在作用中的覆疊排程中變更解析度、x/y 座標、不透明度，甚至是影像本身。

上下文疊加啟動

為便於訪問，如果您希望為播放列表中的特定視頻顯示疊加層，您可以將鼠標懸停在播放列表中的視頻上，然後單擊添加疊加圖標 ( )，UI 將啟動 Add Overlays 模式，其中包含所選視頻的開始時間和持續時間。

筆記

  1. 這將為所選視頻的開始時間創建一個時間表，它不會附加到視頻本身。換句話說，如果視頻由於EPG日程的各種變化而被重新排序（或）刪除（或）開始時間改變，覆蓋日程仍將保持在最初安排的開始時間。
  2. 建議將此添加到視頻上下文中的疊加用於正在運行的頻道而不是處於 DRAFT 或 SCHEDULED 狀態的頻道 - 因為運行頻道比其他狀態的頻道更準確（由 EPG 提供支持）。
  3. 在正在運行的頻道上，此選項可用於當前播放視頻之後的所有視頻。
  4. 此選項不適用於 SSAI 頻道上的廣告佔位符視頻。

圖像處理

圖像以像素為單位疊加在底層源視頻像素上。

如果疊加層大於底層視頻或超出底層視頻的邊緣，則疊加層將不會完全應用於源視頻（它將被裁剪）。這適用於 VOD 資產以及即時區段。
