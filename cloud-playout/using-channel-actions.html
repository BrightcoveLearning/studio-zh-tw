使用頻道動作

根據通道的狀態，在通道設置頁面上可以使用各種通道操作按鈕。通過單擊頻道名稱來訪問頻道操作按鈕以查看頻道詳細信息。下列通道操作按鈕可用:

  • - 激活頻道使其可用於流式傳輸。您必須向頻道添加視頻和/或直播活動，然後單擊 在“激活”按鈕出現之前。
  • - 點擊現在播放開始流式傳輸頻道。此選項僅適用於開始時間即將到來的頻道。頻道將在預定的開始時間自動開始。
  • - 點擊現在停止停止正在運行的頻道。這將停止該頻道並結束直播。如果該頻道正在循環播放，並且具有結束日期/時間，則該頻道將在該時間自動停止。此選項為您提供了一種提前停止它的方法。

通道停止時資源的取消分配

當您停止頻道時，會發生以下情況:

  1. 停止時，Cloud Playout頻道將進入草案狀態，這將使用戶能夠再次修改頻道設置。
  2. 如果有的話，這還將清除鏈接的BC Live事件。
  3. 頻道必須是活性再次，以便資源將被分配並將通道放入預定時間狀態。
  4. 如果為直播流啟用了Cloud Playout頻道，它將顯示一個RTMP攝取的靜態入口點無論通道停止並重新啟動多少次，它在通道的整個生命週期內都將保持靜態。

    通道啟動並運行後，此入口點將能夠開始接受輸入。

  5. 如果由於內部問題，Cloud Playout無法提供靜態RTMP接收點，我們將嘗試顯示基於IP地址的接收點。如果發生這種情況，建議您定期檢查該通道的RTMP接收點是否發生更改。
  6. 筆記: 激活頻道後，Cloud Playout平均需要1-2分鐘才能發布靜態攝取點的新IP地址。因此，如果在激活後的1-2分鐘內立即啟動了某個頻道，並且直播事件恰好是該頻道播放列表中的第一項資產，則可能存在無法解析收錄點的情況。一旦發佈靜態擷取點的 IP 位址，就會變成可解析。
  7. 筆記: 如果通道在首次運行時使用實時輸入運行並停止，然後又關閉了該通道的實時輸入，則現有程序可能仍會存在實時事件，該事件將失敗。如果您決定在停止頻道之後停用直播，您應該先從頻道內容中移除任何即時事件，然後再重新啟動該頻道內容。
