什麼是CDN？

出版商使用CDN來將其媒體廣泛分發給在線觀看者。維基百科將CDN定義為:

內容交付網絡或內容分發網絡（CDN）是跨Internet聯網在一起的計算機系統，它們透明地協作以最頻繁地交付內容，以提高最終用戶的性能，可伸縮性和成本效率。

有許多市售CDN。如果您是Video Cloud Enterprise或Express發布商，Brightcove是您的CDN解決方案。如果您是Video Cloud Enterprise發布商，則可以選擇使用Brightcove CDN或更高版本交付內容Video Cloud支持的任何CDN。

我可以在沒有CDN的情況下交付內容嗎？

如果您期望流量很小或完全控制視頻流量，則可能不需要CDN來提供視頻。例如，某些發布者希望在內部Wiki或專用網絡上分發其內容。您的系統可能不需要外部CDN就可以處理媒體傳遞。如果您想了解有關Brightcove如何支持此設置的更多信息，請聯繫Brightcove客戶支持或您的客戶經理。

我的媒體內容如何傳遞？

有幾種類型的內容交付機制:

  • 流媒體
  • 漸進式下載

流式傳輸（也稱為HLS或DASH）

流視頻是通過流服務器傳遞的，而文件從未下載到查看器的計算機/設備中。觀眾按下播放鍵後，視頻就會開始播放。如果用戶決定轉發或跳至視頻的其他部分，則他/她可以立即執行此操作，並且視頻將從該點開始繼續播放。流媒體的優點之一是帶寬僅用於觀看者已觀看的視頻，因為只有已觀看的視頻部分已交付。客戶端上什麼都沒有保留；一切都在服務器端。

請注意，在 HTML5 視頻播放器時代，所有現代設備和瀏覽器都支持流式傳輸，並且會在可用時使用它（而不是漸進式下載）。這是一件好事，因為它可以更好地管理緩衝並節省帶寬，如果觀眾不觀看整個視頻。

在需要或需要執行以下操作的情況下，流式傳輸很有用:

  • 傳送長文件（10分鐘或更長時間）或高比特率文件（高清）
  • 利用視頻雲多比特率流傳輸功能，該功能執行帶寬檢測並為可用硬件提供最佳質量；您可以將多比特率流媒體與漸進式下載一起使用，但是，帶寬檢測僅在播放開始時發生，並且播放器無法適應查看器帶寬的變化
  • 以更少的帶寬提供更多的流
請注意，您可能會遇到對 RTMP 流的引用。這是較舊的 Flash 串流格式，現在很少使用。

流式傳輸到iOS設備

Apple iOS設備僅支持Apple HTTP Live Streaming（HLS）和HTTP（漸進式下載）來傳送視頻文件。您需要確保您的視頻具有一個或多個可用的HLS格式。App Store規則要求對長格式視頻內容（大於5 Mb或10分鐘）使用Apple HTTP Live Streaming。

流式要點:

  • 通過臨時視頻鏈接保護視頻內容
  • 可以提前播放視頻
  • 有效利用Video Cloud的動態交付功能
  • 讓您以更少的帶寬提供更多的蒸汽
  • 使用的Apple HTTP Live Streaming（HLS）或實時消息協議（RTMP）協議

在實踐中，除非您尋找上述某些獨特功能，否則您幾乎不會意識到內容是流式傳輸還是逐步下載的。

漸進式下載（也稱為PD）

只有非常舊的瀏覽器或設備仍然需要漸進式下載。它現在的主要用途是提供可以由用戶下載和保存的轉譯。

當您使用漸進式下載交付內容時，就像常規網頁或任何其他可下載文檔一樣，文件是通過HTTP請求從標準Web服務器提供的。播放視頻時，視頻文件將下載到查看器的計算機或設備中，並存儲在一個臨時目錄中。當足夠的文件已下載到計算機/設備時，視頻將開始播放。如果觀看者想快進或跳到視頻的另一部分，則只有在已經下載並存儲了視頻的那部分後，他/她才能這樣做。與流視頻相比，漸進式下載確實只有一個一致的好處-您不需要流服務器來交付視頻。可以從任何普通的Web服務器上提供漸進式下載視頻。

漸進式下載對於流量要求不高的業餘愛好者或網站很好，不要介意其內容是否已緩存在查看器的計算機/設備上，以及您是否只需要傳遞較短長度的視頻（不到10分鐘）。需要高級功能並控制其視頻交付的發布者，和/或需要向更大的受眾群體（數百名或更多同時觀看者）顯示視頻的發布者，需要跟踪和報告視頻的使用情況或查看統計信息，或希望提供最佳的交互式播放體驗，將需要流式傳輸視頻。流傳輸還比漸進式傳輸消耗更少的帶寬，因為實際上只觀看了一部分視頻。

漸進式下載要點:

  • 有限的搜尋和導航功能
  • 觀看者可以訪問和重新使用內容
  • 無法有效利用Video Cloud的動態交付功能
  • 安全性較低，內容很容易複製
  • 由於下載了視頻的所有部分，而不僅僅是觀看的部分，因此會佔用更多帶寬

圖像資產如何交付？

無論您對視頻內容使用流式下載還是漸進式下載，都將通過HTTP下載來交付在視頻雲播放器中交付給最終用戶的圖像（例如，縮略圖和視頻靜止圖像）。

比較內容交付: Brightcove託管的視頻資產與BYO相對於遠程資產

Video Cloud支持一些基本的CDN配置，具體取決於您的CDN提供商和設置。選擇是:

  • Brightcove CDN
  • 自帶CDN
  • 遠程資產
  • 多項選擇

Brightcove CDN

您可以選擇使用一個亮灣CDN。Video Cloud利用Akamai，Fastly和其他1級CDN為您的視頻內容提供流式傳輸和漸進式下載交付。通常，使用HLS或DASH流，並且僅在特殊情況下使用逐行下載或提供視頻的可下載副本。

自帶CDN

您也可以選擇使用自己選擇的CDN（自帶CDN或“自帶帶寬"）。在這種情況下，根據與CDN提供商達成的協議，您的視頻將通過漸進式下載（PD）或流式傳輸機制通過視頻雲播放器無縫播放。如果選擇BYO CDN，則可以使用基於視頻雲拉取的內容來將內容提取到您的視頻雲帳戶中，然後將其轉移到您的CDN提供商中，以通過您的視頻雲播放器交付給最終用戶。

遠程資產

最後的選擇是 遠程資產。 在這種情況下，您可能已經將視頻文件存儲在CDN上，並且根本不想使用視頻雲上傳工具來添加新內容。您可以使用 Video Cloud Media 模塊、Dynamic Ingest API 或內容管理系統 API在您的 Video Cloud 帳戶中創建指向由您的 CDN 提供商遠程存儲的基礎視頻資產的視頻。

局限性

使用遠程資產創建視頻時，不能使用圖像捕獲功能為遠程資產視頻創建視頻靜止圖像和縮略圖。可以上傳視頻靜止圖像和縮略圖圖像以用於遠程資產視頻。

有關使用遠程資產的信息，請參閱使用遠程資產視頻。

多項選擇

可以將Video Cloud帳戶配置為同時使用BYO CDN和遠程資產。例如，如果您的CDN上已經有一些視頻內容，則您可能希望對現有內容使用遠程資產以避免再次上傳，而對以後的新上傳使用BYO CDN。如果您需要此設置，請與我們聯繫。您也許還可以創建多個Video Cloud帳戶，每個帳戶都有不同的內容交付策略。

配置內容交付以使用BYO CDN或遠程資產

以下是設置BYO CDN或遠程資產的主要步驟:

  1. 設置一個帶有CDN的帳戶（請參閱以下部分以查看當前受支持的帳戶）。
  2. 如果您有需要遷移的現有視頻內容（從Brightcove或外部CDN），請與我們的服務團隊聯繫。
  3. 請與您的視頻雲客戶經理聯繫以完成設置。

Video Cloud對BYO CDN支持哪些CDN？

Video Cloud支持以下主要CDN用於BYOB CDN內容交付:

  • 赤舞
  • CloudFront
  • 快速
  • 喬丹
  • 聚光燈

可能支持其他BYO CDN，但功能支持會有所不同。與銷售人員聯繫或您的客戶經理以獲取更多信息。

BYO CDN支持的功能

BYO CDN支持以下功能:

  • HTTP / HTTPS
  • 僅HTTPS
  • 簽名清單網址
  • 簽名的內容URL（僅適用於上面列出的CDN）
  • CDN地理限制（如果CDN支持）
  • CDN IP限制（如果CDN支持）
  • 自定義TTL設置（如果CDN支持）

視頻雲報告和BYO CDN或遠程資產

如果您使用BYO CDN或遠程資產，則由於我們未與您的CDN集成，因此您下載的千兆字節將不會出現在Video Cloud Studio Analytics中的任何視頻或播放器中。（您將看到已查看的會議記錄，因為這些記錄是由Brightcove Player報告的。）您應該直接從CDN提供程序獲取這些報告。您可能仍會在“視頻雲"報告中看到針對“非視頻"內容下載的千兆字節，即因視頻雲播放器，圖像等引起的帶寬使用情況。
