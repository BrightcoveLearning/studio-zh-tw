什麼是Video Cloud HLS加密？

使用HTTP實時流加密（HLSe）時，將使用AES 128標準對內容進行加密，並且清單包含指向允許解密HLS內容的密鑰的鏈接。僅憑這一點並不能提供完全的安全性，因為一旦獲得了密鑰，就可以輕鬆地解密和重新分發內容。很少有機制可以保護密鑰，例如通過HTTP或其他令牌身份驗證模型提供密鑰。HLSe確實針對大多數試圖獲取該內容的標準用戶保護該內容，但並未將其視為DRM級別的內容保護。Video Cloud在Brightcove播放器中支持HLS加密。有關Brightcove Player支持的更多信息，請參見Brightcove Player系統要求。還檢查了局限性下面的部分。

Video Cloud Enterprise如何使用HLS加密保護您的內容？

獨立於加密的Apple HTTP Live Streaming（HLS）是一種使用不同比特率的視頻服務協議。Video Cloud支持創建多個再現，這些再現可在網絡帶寬變化和服務波動時在再現之間進行智能切換。HLS本質上將視頻分為一系列小文件下載，每個文件通過HTTP一次加載一個視頻的一小段或一小段。

筆記: 對於長格式視頻（即大於5 Mb或大於10分鐘的視頻），Apple要求使用HLS。若要在 iOS 裝置上播放超過 10 分鐘的影片，不論加密為何，您都必須建立 HLS 轉譯。

Video Cloud支持對Apple HLS的視頻演示文稿進行加密，以便發布者可以保護通過HLS傳送到設備的長格式視頻內容。Video Cloud HLS加密通過將AES添加到我們的標準HLS解決方案中來保護內容。在為Apple HLS實施加密時，Video Cloud會對視頻的每個小文件片段進行加密，並安全地交付處理移交選擇的文件。

支持的功能

除了利用AES規範對電子數據進行加密之外，Video Cloud HLS加密還通過以下方式保護內容:

  • 每個段文件都是加密的
  • Video Cloud交付的HLS清單（.m3u8文件）包含指向每個段的鍵的鏈接
  • 默認情況下，按鍵每10分鐘旋轉一次；您可以指定encryption_key_rotation_period在自定義提取配置文件中。
  • 為啟用了以下功能的帳戶的HLS格式添加加密: 動態投放，只需向Brightcove支持提交請求以啟用該帳戶的HLSe。（對於已經啟用HLSe的帳戶，如果您的帳戶啟用了動態交付，則加密將繼續應用於所有HLS格式。）
  • 要使用舊版提取系統（非動態投放）向您的帳戶中的HLS副本添加加密，請添加encryption_method和encryption_key_rotation_period攝取資料中定義的每個HLS格式的字段: 
    {
  "media_type": "video",
  "reference_id": "ts0",
  "format": "ts",
  "type": "segmented",
  "audio_codec": "aac",
  "audio_bitrate": 64,
  "video_codec": "h264",
  "encryption_method": "aes-128",
  "encryption_key_rotation_period": 10,
  "video_bitrate": 450,
  "decoder_bitrate_cap": 771,
  "decoder_buffer_size": 1028,
  "keyframe_rate": 0.5,
  "max_frame_rate": 30,
  "width": 480,
  "height": 270,
  "h264_profile": "baseline"
筆記: 視頻加密本身並不是內容保護的強大形式。如果內容安全性對您的組織至關重要，則應僱用DRM保護。如需搭配 HLS 加密和 DRM 保護的詳細資訊，請聯絡您的客戶經理，以便在任何地方播放內容的安全。

HLS加密實施後會發生什麼

視頻雲HLS加密可提供安全的多比特率編碼，其中，每種再現形式以及每個再現形式的每個片段都以多種方式受到保護。上傳和加密視頻的第一個副本後，可以在桌面和移動設備上播放HLS加密的視頻。一旦實施，此後上傳的所有視頻都將使用HLS加密進行保護。Video Cloud HLS加密對設備上的視頻播放沒有任何可檢測到的更改。Video Cloud HLS加密僅影響視頻文件的HLS格式，對MP4格式沒有影響。

局限性

  • HLS加密適用於您帳戶中的所有非DRM HLS格式。如果您有要傳播的促銷視頻或其他視頻而沒有加密，則可以將其上傳到其他視頻雲帳戶，而不啟用HLS加密。
  • 對於使用舊版攝取系統攝取的視頻，在實施HLS加密之前，Video Cloud不支持加密上傳到Video Cloud的HLS副本。先前上傳的HLS內容未加密。你必須重新轉碼視頻在HLS加密之前上載以保護它們。這確實不是適用於使用動態投放系統提取的視頻。
  • Video Cloud在“媒體"模塊中未顯示用於標識HLS加密視頻的指示器。
  • 如果用戶在Apple設備上播放經過HLS加密的視頻，然後在TTL過期後嘗試重播，則播放將無法開始，並且不會向用戶提供警報消息。
  • 使用Brightcove Player時，IE8或IE9不支持HLS播放。這些設備將退回到MP4。
  • HLSe支持Android SDK並且在帶有VisualOn組件的舊版本中將受支持。HLSe還應該在HTML5播放器的4.x上工作。有關Android設備上HLSe支持的更多信息，請參見Android支持的媒體格式。
  • 對於HTML5播放器，HLSe支持完全由基礎操作系統/設備確定。
  • 如果您選擇終止 Video Cloud HLS 加密，將從帳戶中的所有視頻中刪除加密 - 無需重新轉碼。
  • 當前，使用Brightcove Native SDK for Android，iOS或tvOS的脫機播放不支持HLSe內容。
