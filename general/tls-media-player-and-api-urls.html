介紹

傳輸層安全性（TLS）通過加密來保護通過Internet傳輸的數據。（TLS取代了一種稱為SSL的較早的加密協議。）該過程對用戶基本上是透明的。通過Web傳遞的數據可以採用加密或不採用加密的方式進行傳遞，因此，由客戶端（例如，代理人）向服務器發送信號，告知應該對數據進行加密。通常，這是通過對URL使用HTTPS協議而不是HTTP來完成的。

Video Cloud按照以下條款支持TLS用於所有媒體傳輸:

  • 那些使用Brightcove CDN的人可以聯繫Brightcove支持為其配置帳戶HTTPS + HTTP傳送數據，或用於僅HTTPS交貨。
  • 使用其他CDN（有時稱為BYO CDN）的用戶可以從其CDN請求這些選項（如果支持）。

是否需要TLS主要取決於部署視頻的站點/頁面是否通過HTTPS加載-如果需要，則還必須通過HTTPS；否則，它們可能無法在查看器的瀏覽器中加載或生成警告。

TLS支持的變化

遵循互聯網安全專家的建議，Brightcove更改了對TLS版本的支持，如下所述。

有什麼變化？

Video Cloud逐漸結束了對TLS版本1.0或1.1的支持。使用這些版本的客戶必須盡快遷移。

TLS 1.0和1.1的支持已關閉，原因是:

  • 圖庫API
  • 交付系統API
  • 播放器管理 API

為什麼Brightcove棄用了較舊的TLS版本？

客戶數據的安全性是Brightcove的頭等大事。這些變化正在我們的行業中不斷發展，並且我們的許多技術合作夥伴和同行已經採取了禁用早期TLS的措施。

我怎麼知道我們是否準備好進行此更改？

由於不建議使用TLS 1.0 / 1.1，因此與視頻雲平台的連接將需要使用TLS 1.2加密協議（或更高版本）。我們希望影響不會太大，但是存在兩個風險領域:

  1. 您應確保與Video Cloud API的所有連接（例如，從Python腳本到CMS API的調用）均來自支持TLS 1.2或更高版本的平台。這意味著應該從兼容的平台託管自動化流程。
  2. 如果您的用戶使用不受支持的瀏覽器，禁用了相關的加密協議或使用了不受支持的設備，則可能會受到影響。

一般而言，影響應僅限於舊版設備和瀏覽器，這些設備和瀏覽器僅占我們用戶的一小部分。請參閱下面的示例，了解不支持的客戶端和設備。

不支持的平台（播放器/ SDK）

如前所述，“明文" HTTP連接不受更改的影響。以下列出了不支持使用TLS 1.2的HTTPS連接的用戶代理列表。

操作系統/瀏覽器組合

  • Windows 7 / Chrome 29（或更早的Chrome版本）
  • Windows 7 / Firefox 10.10.12，Firefox 17.0.7，Firefox 24.2.0，Firefox 22，Firefox 24
  • Windows 8 / Firefox 26
  • Windows XP / IE 6
  • Windows Vista / IE 7
  • Windows Phone 8.0 / IE 10
  • Windows 7 / Opera 12.15，Opera 15，Opera 16
  • OS X 10.6.8 / Safari 5.1.9
  • OS X 10.8.4 / Safari 6.0.4
  • Android 4.3或更早版本

啟用TLS 1.2

在某些情況下，可能支持但禁用TLS 1.2。要在Windows Web瀏覽器上啟用TLS 1.2協議，請參閱下文。

  • Microsoft Internet Explorer
  • 谷歌瀏覽器
  • 火狐瀏覽器
  • 歌劇

Microsoft Internet Explorer

  1. 打開Internet Explorer。
  2. 在菜單欄中，單擊工具> Internet選項>高級選項卡:
  3. 向下滾動到“安全"類別，手動選中“使用TLS 1.2"的選項框。
  4. 單擊確定。
  5. 關閉瀏覽器，然後重新啟動Internet Explorer。

谷歌瀏覽器

  1. 打開谷歌瀏覽器。
  2. 單擊Alt-F，然後選擇設置。
  3. 向下滾動並選擇顯示高級設置。
  4. 向下滾動到“網絡"部分，然後單擊“更改代理設置"。
  5. 選擇高級選項卡。
  6. 向下滾動到“安全"類別，手動選中“使用TLS 1.1"和“使用TLS 1.2"的選項框。
  7. 單擊確定。
  8. 關閉瀏覽器，然後重新啟動Google Chrome。

火狐瀏覽器

  1. 打開Firefox。
  2. 在地址欄中，鍵入about: config，然後按Enter。
  3. 在搜索字段中，輸入tls。查找並雙擊以下條目security.tls.version.max。
  4. 將整數值設置為3以強制執行TLS 1.2協議。
  5. 單擊確定。
  6. 關閉瀏覽器，然後重新啟動Mozilla Firefox。

歌劇

  1. 打開Opera。
  2. 單擊Ctrl加F12。
  3. 向下滾動到“網絡"部分，然後單擊“更改代理設置"。
  4. 選擇高級選項卡。
  5. 向下滾動到“安全"類別，手動選中“使用TLS 1.1"和“使用TLS 1.2"的選項框。
  6. 單擊確定。
  7. 關閉瀏覽器，然後重新啟動Opera。
