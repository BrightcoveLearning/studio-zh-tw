可以為視頻配置多個音軌。多個音軌可用於:

  • 播放不同語言的同一視頻 
  • 為視障人士提供音頻和說明
  • 提供立體聲與多聲道（環繞聲）音頻

HLS V4 +和DASH支持多個音軌-帶有和不帶有DRM保護。具有多個音軌的視頻將顯示一個耳機圖標 () 在控制欄中。單擊該圖標後，觀看者可以在音軌之間進行選擇。

筆記:
  • 多個音軌只能與為以下內容設置的視頻一起使用動態投放。此外，必須為您的帳戶啟用多個音頻軌道。請與您的客戶經理聯繫以獲取詳細信息。
  • 最初攝取視頻後，是否在標題中添加了音軌；處理音軌時，將選擇發布者帳戶的默認攝取配置文件。我們建議以與原始視頻標題相同的攝取配置文件來處理音軌，以確保使用相同的音頻比特率/格式。有關設置默認攝取配置文件的更多詳細信息，請參閱管理攝取配置文件。
  • 使用動態投放攝取配置文件上傳視頻後，視頻的源音頻將根據攝取配置文件進行轉碼，並顯示在音軌媒體模塊中的視頻部分。默認情況下，音頻語設定為英語與一個變體類型主要的。可以通過Brightcove支持更改默認值。然後可以添加其他音軌。
  • 當您向視頻中添加其他音軌時，它們必須都具有相同的比特率/再現配置文件，以確保在HLS中成功播放（DASH播放仍將起作用）。
  • 當您將音軌添加到視頻中，然後在第一個音軌完成上傳之前添加一個副本時，其中一個音軌可能會在無提示的情況下被破壞，導致視頻播放失敗。確保等到每首曲目都完全上傳後再上傳另一首。

配置音軌屬性

音軌文件可以由Brightcove上載並託管，也可以位於Internet可訪問的URL上。要查看和編輯視頻的音軌屬性，請按照下列步驟操作:

  1. 打開媒體模塊。
  2. 點擊視頻標題鏈接，然後找到音軌部分。將顯示源視頻的音軌。
  3. 點擊編輯或雙擊該部分。
  4. 點擊添加音軌。
  5. 選擇是從本地文件系統還是從外部URL上傳音軌文件。
    • 上傳文件 - 點擊瀏覽並從本地文件系統中選擇音軌文件。該文件將被上傳到Video Cloud。
    • 網址上傳 -輸入音頻軌道文件所在的公共URL，然後單擊上載。該文件將被上傳到Video Cloud。
      筆記: 帶有圓括號，問號，引號，空格和其他特殊字符的音軌文件名在上載時將自動失敗。布萊特灣建議使用不含特殊字元的檔案名稱。
  6. 這語字段應包含語言代碼指定軌道文本數據的語言。
  7. 選擇變體文本軌道。這變體描述了音軌的種類。選項包括:
    • 主要的 - 主音軌，通常是混入視頻文件的音軌
    • 備用 - 備用音軌
    • 評論 - 在視頻軌道上提供評論的音軌
    • 配音 - 包含不同語言的口語配音版本的曲目
    • 描述性的 - 曲目以某種方式描述了視頻內容
  8. 點擊添加音軌保存音軌信息。
  9. 點擊添加音軌添加另一個音軌或保存保存音軌。處理完成後，新添加的音軌將出現在列表中。
    筆記: 對音軌內容所做的變更可能需要 15 分鐘才會出現在已發佈的播放器中。

要刪除音軌，請單擊刪除圖標 () 旁邊的音軌。請注意，在更新UI且音軌不再出現之前可能會有延遲。

發布具有多個音軌的視頻時，耳機按鈕 () 將出現在播放器的 chrome 上。點擊按鈕選擇一個音軌。

設置默認音軌

上載視頻時，源音頻將被設置為主音軌（默認）。要將音頻軌道設置為默認軌道，請按照下列步驟操作:

  1. 點擊編輯或雙擊音軌部分。
  2. 單擊當前為默認音頻軌道的軌道。
  3. 取消選中默認選項。
  4. 單擊將被設置為默認音頻軌道的軌道。
  5. 檢查默認選項。
  6. 點擊更新音軌。

局限性

使用多個音軌時，以下限制適用

音頻主機未存儲
如果您重新轉換視頻母帶中的視頻，則其他音軌將丟失，必須重新添加
必須使用純音頻文件
音軌必須是不帶視頻音軌的純音頻文件
HLSv3，HLS和音頻和視頻在同一段中
  • 播放API將不會返回HLSv3清單
  • 所有HLS清單將包括解混合的視頻/音頻
流暢的串流
平滑流URL將不可用。
社會分佈
無法選擇將用於分配的音軌。視頻源中包含的音軌（混合在音頻中）將始終使用。
音頻軌道的順序超出一個“默認"軌道
  • 您可以選擇默認音軌內容管理系統 API每個標題通過設置is_default場到true
  • 還有一個帳戶默認值，可以由支持人員設置
  • 這只會影響HLS清單中的“默認"軌道
  • 不能進行其他訂購
從單一來源攝取多個音軌
我們支持來自“.master"中提供的源的一個音軌，並在“.audio_tracks.masters"下為每個來源提供一個。
僅包含音頻的視頻的DRM保護
添加視頻軌道後，將啟用DRM保護。
最終用戶友好標籤
我們不支持音軌的自定義標籤。如果需要，您將需要通過Player API在客戶端進行更改。
在某些情況下，切換軌道可能會導致Brightcove Player變得不穩定
  • 在下載所有音頻片段之前進行音軌切換
  • 當使用Silverlight插件播放視頻時（IE版本小於10，或Windows版本小於8的任何IE）-多條音軌不是在Silverlight中受支持。
  • 如果音頻和視頻的持續時間不同，則較短的播放器用完時播放器可能會停止播放。
視頻“持續時間"
該視頻duration如果音軌的持續時間不同，則目錄/播放API報告的錯誤可能不正確。
