媒體共享使發布者可以一次上傳視頻，然後將其共享到其他帳戶。這些帳戶可以是您自己的公司帳戶，也可以是其他發布者的帳戶。然後可以在其他視頻雲帳戶中獨立使用共享的視頻。當發布者想要保留視頻內容的主帳戶，然後將視頻共享給組織的其他部門或子公司時，通常使用媒體共享。

筆記:
  • 媒體共享僅適用於Video Cloud Enterprise客戶。如果您有興趣升級您的Video Cloud帳戶，請聯繫Brightcove以獲取更多信息。
  • 媒體共享可以是雙向的: 每個帳戶可以是另一個帳戶的主帳戶和附屬帳戶。

媒體共享也可以使用CMS API以編程方式實現。有關更多信息，請參見使用CMS API進行媒體共享。

媒體共享術語

本主題將使用以下術語:

  • 掌握 -創建原始視頻的帳戶。主人擁有內容，並負責設置，管理和向會員提供內容。
  • 會員關係 -正在接收視頻的帳戶。聯屬會員可以接受母版分享給它的內容。

媒體共享限制

  • 共享使用DRM保護的視頻是不是支持的。支持將視頻從未啟用DRM的帳戶共享到已啟用DRM的帳戶，但是共享的視頻將不是打包為DRM。

  • 對於使用“將實時流轉換為VOD"選項創建的視頻，共享將失敗。它適用於通過剪輯實時流製作的視頻。

  • 僅在可以共享原始視頻的情況下，才可以下載共享的視頻以供離線查看。該屬性是從原始視頻繼承的，不能僅針對共享視頻進行更改。

  • 當使用SSAI播放共享視頻時，替換SSAI Macro將使用父視頻而不是子視頻中的元數據。如果父視頻標記為Advertising='Free'，即使子視頻被標記為Ad Supported。

管理媒體共享關係

在帳戶之間共享視頻之前，必須將媒體共享設置配置為管理>媒體共享設置。請注意，只有帳戶管理員可以訪問“管理"菜單。如果您希望與其他帳戶共享視頻，則必須添加會員，會員帳戶必須接受這種關係。主賬戶和會員賬戶可以隨時終止關係。終止媒體共享關係才不是從會員帳戶中刪除所有共享的視頻。如果要從會員帳戶中刪除共享的視頻，則需要從您自己的（主）帳戶中刪除共享的視頻或打開視頻屬性，請轉到媒體分享部分，然後單擊刪除視頻會員帳戶的圖標。

有關配置媒體共享設置的信息，請參閱管理媒體共享設置。

分享影片

默認情況下，帳戶中的所有用戶都可以訪問媒體模塊，從而可以共享視頻。要共享視頻，請按照以下步驟操作:

  1. 打開媒體模塊，選擇一個或多個視頻，然後單擊更多>分享。
    筆記: 當共享大量視頻時，Brightcove建議一次共享最多50個視頻。一次共享大量視頻可能會導致Studio中出現超時錯誤。如果您需要分享大量影片，應該使用CMS API。
  2. 選擇要共享視頻的會員帳戶，然後單擊分享。搜索字段可用於搜索帳戶ID。搜索字段支持模糊搜索，因此輸入ID的一部分將返回匹配項。

可以重新共享視頻（將以前共享的視頻共享到同一帳戶）。這樣做會將聯盟會員帳戶對視頻元數據所做的任何更改替換為主帳戶（共享）中的值。重新共享視頻時請多加註意。

查看分享的視頻

會員帳戶可以配置接受方式對於每個主帳戶。當。。。的時候接受方式是汽車，共享的視頻將顯示在“媒體"模塊中會員帳戶的視頻列表中。當。。。的時候接受方式是手動的， 一種股票待定顯示共享視頻數量的鏈接將出現在“媒體"模塊中。

筆記: 如需有關設定接受模式的資訊，請參閱管理媒體共用設定。

點擊股票待定鏈接將顯示共享視頻的列表。

從待定股票列表中，您可以:

  • 拒絕 -拒絕共享視頻並且不將其包含在您的帳戶中。拒絕共享的視頻不會阻止主服務器將來再次將其共享給您的帳戶。
  • 接受 -將共享視頻接受到您的帳戶中。

快速瀏覽可用於打開面板以查看視頻屬性。接受共享視頻後，該視頻將不再顯示在“媒體"模塊的“待共享"列表中，而是顯示在“所有視頻"視圖中。

媒體模塊中的共享視頻

共享的視頻將在視頻縮略圖的右上角顯示一個共享圖標，以指示該視頻已共享到一個帳戶中或從一個帳戶中共享出來。

視頻已分享到一個帳戶

將鼠標懸停在共享圖標 () 將顯示共享詳細信息。

視頻已分享到另一個帳戶

將鼠標懸停在共享圖標 () 將顯示共享詳細信息。

主帳戶可以使用來查看共享給會員帳戶的視頻的狀態媒體分享視頻屬性頁的“部分"。

錯誤可能還會在這裡出現。例如，如果實施自定義字段匹配如果選中，則附屬帳戶必須具有與主帳戶相同的自定義字段，否則任何共享都會失敗。

刪除共享視頻

從主帳戶刪除共享視頻後，會顯示一條消息，提醒您刪除共享視頻會將其從與之共享的所有聯屬會員帳戶中刪除。

共享的視頻屬性

將視頻共享到會員帳戶時，以下視頻屬性將與共享視頻一起復制:

  • 名稱
  • 簡短描述
  • 參考ID（如果您與之共享的任何會員帳戶已經具有具有相同參考ID的視頻，則共享操作將失敗，並顯示一條錯誤消息。您可以通過設置參考ID命名約定來避免這種情況，以防止會員使用相同的參考ID。）
  • 相關鏈接（鏈接文本和鏈接URL）
  • 標籤
  • 視頻投影
  • 縮略圖和海報圖像-默認情況下，主視頻的海報和縮略圖圖像將在會員帳戶中使用。如果對主帳戶中的發帖人和/或縮略圖進行了更改，則這些更改將反映在會員帳戶中除非聯屬會員帳戶之前已生成新的海報或縮略圖圖像。一旦會員帳戶更改了海報或縮略圖，則對主帳戶中的海報/縮略圖進行的任何更改將不會傳播到會員帳戶。
  • 文本軌道-關聯公司無法刪除或編輯共享視頻的文本軌道屬性
  • 音軌-關聯公司無法刪除或編輯共享視頻的音軌屬性
  • 廣告（免費/廣告支持）
  • 廣告鍵
  • 安排開始日期和結束日期
  • 地理限制-發布者應確保主帳戶和會員帳戶的地理限製配置方式相同。
  • 提示點及其關聯的元數據
  • 自定義字段-如果主帳戶配置為實施自定義字段匹配，會員帳戶必須具有與主帳戶相同的自定義字段，否則任何共享都會失敗。
筆記: 如果自訂欄位是清單類型，則自訂欄位在兩個帳戶中必須具有相同的選項。

共享視頻具有共享主視頻時具有的這些視頻元數據字段的所有值。但是，從那時起，主副本和附屬副本將成為獨立的實體:

  • 使用媒體共享時，不復制視頻文件，僅複製視頻元數據。共享的視頻指向您帳戶中的原始視頻文件。
  • 如果從主帳戶中刪除了該視頻，則該視頻將從所有聯屬會員帳戶中刪除。
  • 聯盟會員對視頻所做的任何更改均不會影響原始或任何其他共享副本。如果會員刪除了共享視頻，該視頻將從會員帳戶中刪除，但仍保留在主帳戶和所有其他會員帳戶中。
  • 關聯公司無法使用媒體模塊刪除或更新文本軌道。
  • 會員通過共享收到視頻時，該視頻處於活動狀態。即使該視頻未在主帳戶中處於活動狀態也是如此。
  • 如果會員接受視頻並且缺少必填字段，則該視頻將處於無效狀態。
  • 接收共享視頻的會員帳戶無法將該視頻共享給另一個帳戶。只有主帳戶可以共享視頻。
  • 每個帳戶都要單獨計費，以從該帳戶中播放共享視頻（視頻播放和帶寬使用）。
  • 報告共享視頻是特定於每個帳戶的，並不匯總有關每個帳戶中視頻的每個副本的信息。

可以重新共享視頻（將以前共享的視頻共享到同一帳戶）。這樣做會將聯盟會員帳戶對視頻元數據所做的任何更改替換為主帳戶（共享）中的值。重新共享視頻時請多加註意。

共享視頻副本

可以將視頻共享到會員帳戶前轉碼已完成。接收共享視頻的會員帳戶將有權訪問在主帳戶中創建的副本。如果源帳戶中的視頻被重新轉換，或者源文件被更改，則這些更改將在接收共享視頻的會員帳戶中發生。

關聯帳戶無法重新轉換或替換共享視頻的源文件。

共享媒體和賬單

我們經常會問到帳戶之間共享的媒體如何影響帳單的問題。以下是一些場景，這些場景說明瞭如何處理髮布者之間共享的內容的計費。

場景1

  • 在Brightcove CDN上具有內容的Pub A和Pub B
  • 酒館A分享到酒館B
  • 酒吧B向Brightcove支付流和帶寬

方案2

  • 使用BYO CDN的Pub A
  • Brightcove CDN或BYO CDN上的Pub B（無關緊要）
  • 酒館A分享到酒館B
  • 酒吧B向Brightcove支付了流費；客棧A支付自己的CDN帶寬

場景3

  • 在Brightcove CDN上包含內容的Pub A
  • 使用BYO CDN的Pub B
  • 酒館A分享到酒館B
  • 假定正在傳遞計費參數，酒吧B將向Brightcove支付流和帶寬的費用。我們已經看到由於創建自定義視頻播放器而未正確傳遞計費參數的情況下，計費參數未傳遞的情況。

方案4-必須從Origin獲取內容

我們傳遞帳戶ID  在每個請求中 這樣，每個帳戶都將為其自己的原始訪問付費。所以如果共享者帳戶經常錯過其CDN緩存，必須返回原始內容，然後共享者將為此付費。如果共享者帳戶經常錯過其CDN緩存，必須返回原始內容，然後共享者帳戶為此付費。不管CDN配置方案如何，原始訪問的費用都是分開的。

方案5-圖片

默認情況下，使用動態交付時，將為與共享資產關聯的所有圖像流量向主帳戶（圖像的創建者）收費。如果會員帳戶上傳或捕獲共享內容的新圖像，則圖像流量將被關聯併計入會員帳戶。
