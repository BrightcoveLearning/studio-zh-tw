廣告屬性用於為Brightcove Player啟用客戶端和/或服務器端廣告:

  • 客戶端廣告支持針對HTML5版本3的Google交互式媒體廣告（IMA）。這樣一來，您就可以為播放器請求和跟踪VAST廣告。有關Google IMA的詳細信息，請參見使用IMA HTML5 SDK版本3文檔。
  • 服務器端廣告將廣告直接插入視頻內容流中，因此它可以在任何可以播放流視頻的設備上使用。如果沒有SSAI，則如果瀏覽器中存在廣告攔截器，則客戶端廣告（例如使用VAST，VMAP和VPAID的IMA或FreeWheel）將無法播放。要了解有關SSAI的更多信息，請參見Video Cloud SSAI概述文檔。請注意，除非您的帳戶已啟用SSAI，否則服務器端選項將不會出現。

測試廣告服務器（客戶端廣告）

在配置客戶端廣告屬性時，將需要廣告服務器URL。Brightcove建議您驗證廣告服務器URL的有效性。我們發現，有關廣告無法播放的大多數問題是由於廣告代碼本身的問題引起的。驗證廣告代碼是否有效應確保廣告正常播放。Google提供了一個Video Suite檢查器，可用於驗證廣告代碼。要驗證您的廣告代碼，請按照以下步驟操作。

  1. 打開視頻套件檢查器頁。
  2. 將廣告代碼網址粘貼到輸入類型表單輸入字段。
  3. 點擊測試廣告。
  4. 驗證您是否看到了您的廣告。如果您的廣告代碼在此測試環境中無法正常運行，則無法與Brightcove Player一起使用。

廣告屬性用於將Brightcove Player與適用於HTML5版本3的Google交互式媒體廣告（IMA）集成。這樣一來，您就可以為播放器請求和跟踪VAST廣告。有關Google IMA的詳細信息，請參見使用IMA HTML5 SDK版本3文檔。

配置客戶端廣告屬性

要為播放器配置客戶端廣告，請按照下列步驟操作:

  1. 點擊玩家們在導航標題中打開“播放器"模塊。
  2. 單擊您要配置的播放器的鏈接。
  3. 點擊廣告在左側導航菜單中。
  4. 查看啟用客戶端（IMA）。
  5. 輸入廣告代碼。
  6. 選擇要求廣告環境。
    • 負載 -播放器加載後立即請求廣告（這通常是DFP / VPAID的最佳體驗）
    • 進行中-第一個廣告請求被延遲，直到開始播放
    • 一經請求 -所有廣告請求都將使用player.ima3.adrequest()方法
    • 在提示點上-在視頻中遇到廣告提示點時發出廣告請求
  7. 選擇VPAID模式。VPAID模式用於在IMA廣告中啟用VPAID 2支持。
    • 已啟用-在其他網域的iframe中播放VPAID廣告
    • 不安全-在具有相同域的iframe中播放VPAID廣告
    • 殘障人士- VPAID廣告引發錯誤
  8. 設置超時價值--播放廣告之前等待廣告初始化的最長時間（以毫秒為單位）。
  9. 查看硬超時以防止出現加載緩慢的廣告。取消選中此選項可能會導致加載緩慢的廣告中斷視頻播放。
  10. 設置最大重定向數。該參數指定在拒絕後續重定向並終止廣告加載之前，重定向的最大數量。重定向的數量直接影響等待時間，從而影響用戶體驗。
  11. 選擇IMA 插件版本使用。
  12. 點擊保存保存更改。

IMA插件確實支持UI中不可用的其他屬性。如果需要，請修改IMA插件JSON以包括您可能需要的任何其他選項。有關IMA插件特定選項和廣告框架選項的詳細信息，請參閱使用IMA3插件做廣告開發人員文檔。

配置服務器端廣告

使用服務器端廣告時，您必須創建一個或多個廣告配置。廣告配置定義了SSAI播放的各個方面，包括廣告調用，信標和其他配置選項。一個帳戶可以具有多個配置，當前不能在多個帳戶之間共享配置。管理員界面中的服務器端廣告設置菜單用於創建和管理廣告配置。有關信息，請參閱配置服務器端廣告設置。

要為播放器配置客戶端廣告，請按照下列步驟操作:

筆記: 只有在伺服器端廣告啟用帳戶時，才會顯示伺服器端廣告選項。
  1. 查看啟用服務器端（SSAI）。
  2. 使用選擇配置下拉菜單，選擇要與此播放器關聯的廣告配置。
  3. 檢查啟用廣告信息疊加層如果您想在廣告上展示重疊式廣告，請選中該複選框。這包括“了解詳情"和廣告倒計時重疊式廣告。
  4. 點擊保存。
